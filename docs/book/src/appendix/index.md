# Appendix

The appendix publishes repository source-of-truth documents alongside the
task-oriented guide. These copies are generated by `scripts/build-book.sh` on
every build; edit the canonical files in `docs/` or rerun conformance instead
of editing the copies under `docs/book/src/appendix/`.

## Behavioral contracts

[Behavioral Contracts](contracts.md) records what the implementation actually
guarantees: broadcast retention and cursor driving, tool execution order,
three cancellation layers, approval signature boundaries, downloader/SSRF
policy, realtime lifecycle, and the normative C ABI v1 contract.

Use it when correctness depends on a lifetime, race, error-as-data rule, or
binary compatibility promise. Open hardening items are named explicitly
rather than silently presented as solved.

## Porting guide and fidelity ledger

[Porting Guide & Fidelity Ledger](porting-guide.md) maps upstream TypeScript
concepts into Zig 0.16: tagged unions, vtables, `std.Io`, pull streams,
allocators, HTTP/SSE, provider construction, C ABI shapes, and test strategy.

Section 18 is the authoritative ledger for intentional differences. The
differential harness requires every expected deviation to reference that
ledger before it can receive `LEDGERED` status.

## Roadmap

[Roadmap](roadmap.md) preserves the phase dependency order, acceptance gates,
completion notes, and remaining release-readiness work. It is also the source
for the distinction between a completed in-tree ABI v1 policy and the tagged
artifact matrix that cannot start until the first release tag exists.

## Conformance report

[Conformance Report](conformance-report.md) is copied from
`conformance/.artifacts/report.md` when present. Documentation CI runs the
differential suite immediately before the copy, so the deployed report is
per-commit evidence. A local book build without artifacts gets an explanatory
stub, never an old checked-in table.

## Editing and links

Each generated document begins with a source header and is ignored by Git.
The mdBook edit link for a generated page points at its generated location,
but the header identifies the canonical source to change. The guide chapters
cross-link to these documents wherever a concise explanation would otherwise
duplicate normative detail.

For implementation source, tests, and current status, return to the
[repository](https://github.com/mattneel/ai.zig). For a first application,
return to [Getting Started](../getting-started.md).
