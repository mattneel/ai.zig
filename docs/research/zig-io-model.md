# zig-io-model

## Summary
Zig 0.16 replaces all blocking/nondeterministic stdlib functionality with a runtime-dispatched `std.Io` interface: a 2-word value struct (userdata + *const VTable) whose ~100-entry vtable covers async/await/cancel tasks, groups, batches, futex, fs, net, process, time, and entropy. Implementations: Io.Threaded (feature-complete blocking thread pool with signal-based cancellation via SIG.IO/EINTR and a per-thread atomic cancellation state machine), and experimental fiber-based Evented backends (Uring on Linux — no networking yet — Kqueue, Dispatch), plus Io.failing. Concurrency model: `io.async(fn, args)` returns a generic Future(T) (may run inline; infallible), `io.concurrent` guarantees parallelism or error.ConcurrencyUnavailable; Future has await(io)/cancel(io); Io.Group manages N tasks with collective await/cancel; Io.Select races tasks into a tagged-union queue; Io.Queue(T) is the MPMC channel; Batch/Operation is the zero-allocation layer with operateTimeout. Cancellation is cooperative: error.Canceled is in nearly every error set, one-shot per request, re-armed with io.recancel(), maskable with swapCancelProtection. Reader/Writer are non-generic vtable structs with the buffer stored in the interface (stream/discard/readVec/rebase; drain/sendFile/flush/rebase), composed by struct-embedding + @fieldParentPtr with error side-channels on concrete impls. Io.net provides IpAddress.connect/listen → Stream with Stream.Reader/Writer adapters, HostName.lookup/connect implementing cancel-correct happy-eyeballs using async+Queue+Group, and std.http.Client + std.crypto.tls.Client demonstrate the full streaming HTTP+TLS stack an SDK should mirror. Time uses i96-nanosecond Timestamp/Duration, typed Clock, and a Timeout union (none/duration/deadline). Libraries accept Io by value or store it on a context struct; main() owns construction via std.process.Init.

## Key findings
### Io interface shape
std.Io is a 2-word value struct: `userdata: ?*anyopaque, vtable: *const VTable`. VTable (Io.zig:51-255) is ~100 function pointers grouped: task ops (async/concurrent/await/cancel, groupAsync/groupConcurrent/groupAwait/groupCancel), cancellation (recancel, swapCancelProtection, checkCancel), futex (futexWait/futexWaitUncancelable/futexWake), batch (operate, batchAwaitAsync, batchAwaitConcurrent, batchCancel), dir* (~26 fns), file* (~25 fns incl. memory map), process*, now/clockResolution/sleep, random/randomSecure, net* (netListenIp, netAccept, netBindIp, netConnectIp, netListenUnix, netConnectUnix, netSocketCreatePair, netSend, netRead, netWrite, netWriteFile, netClose, netShutdown, netInterfaceNameResolve, netInterfaceName, netLookup). Every vtable fn takes userdata first. Io.failing (Io.zig:2512) is a complete no-op/error implementation useful as a template for custom Io impls.

### async/concurrent/Future exact signatures
`pub fn async(io: Io, function: anytype, args: std.meta.ArgsTuple(@TypeOf(function))) Future(ReturnType)` — infallible; impl MAY run the fn inline before returning (vtable.async returns ?*AnyFuture; null = result already populated, await is no-op). `pub fn concurrent(io, function, args) ConcurrentError!Future(R)` — guarantees a unit of concurrency or fails with error.ConcurrencyUnavailable. `Future(Result)` = struct { any_future: ?*AnyFuture, result: Result } with `await(f: *@This(), io: Io) Result` and `cancel(f: *@This(), io: Io) Result`; both idempotent, not threadsafe, and release task resources. Type erasure at the vtable: context bytes + alignment + `start: *const fn(context: *const anyopaque, result: *anyopaque) void`. Idiom: `var fut = io.async(f, .{args}); defer if (fut.cancel(io)) |res| res.deinit() else |_| {};` — cancel-in-defer is THE resource cleanup pattern.

### Group / Select / Queue
Io.Group (Io.zig:1218) = struct { token: std.atomic.Value(?*anyopaque), state: usize }, `pub const init`. `group.async(io, function, args) void` and `group.concurrent(io, function, args) ConcurrentError!void` — function return type must coerce to Cancelable!void; `group.await(io) Cancelable!void`, `group.cancel(io) void` (idempotent, awaits all). O(1) spawn overhead; task resources freed when each task returns. Io.Select(comptime U: union) wraps Group + Queue(U): `select.async(comptime field, function, args)`, `await(s) Cancelable!U` (returns next completed result as tagged union), `awaitMany(buffer, min)`, `cancel(s) ?U` (loop until null to drain resources), `cancelDiscard`. Io.Queue(T) wraps TypeErasedQueue: MPMC ring buffer over caller-provided `[]T`, ops `put(io, elems, min)`, `putAll`, `putOne`, `get(io, buf, min)`, `getOne`, `close(io)`, plus *Uncancelable variants; blocked putters/getters park on Io.Condition; close() causes error.Closed for puts, drains buffer then error.Closed for gets. Partial progress + cancel → returns count and calls io.recancel() to re-arm.

### Cancellation semantics
`pub const Cancelable = error{Canceled}` appears in nearly all Io error sets — every Io call with it is a cancellation point. Requesting cancel (Future.cancel/Group.cancel/Batch.cancel) makes exactly the NEXT cancellation point in the task return error.Canceled (one-shot); ignoring it is a bug unless you are the canceler. Three handling options: (1) propagate; (2) `io.recancel()` re-arms then don't propagate; (3) `const old = io.swapCancelProtection(.blocked); defer _ = io.swapCancelProtection(old);` makes error.Canceled unreachable in a region (CancelProtection = enum(u1){unblocked, blocked}). `io.checkCancel() Cancelable!void` is a pure cancellation point for CPU-bound loops. cancel==await plus request: the task may still have completed successfully, so canceled tasks returning resources must be cleaned up by the canceler. Group.async fns returning error.Canceled are a propagation boundary (swallowed).

### Threaded implementation (blocking)
Io.Threaded (18.9k lines, feature-complete): growing detached thread pool. Fields: allocator (must be threadsafe; used only for async/concurrent/group closures), run_queue, async_limit/concurrent_limit (Io.Limit), busy_count, worker_threads list. `init(gpa, InitOptions{stack_size, async_limit=?cpus-1, concurrent_limit=.unlimited, argv0, environ, disable_memory_mapping})`, `deinit`, `pub fn io(t: *Threaded) Io`. `init_single_threaded` const and `global_single_threaded` ptr: no concurrency, no cancel, Allocator.failing. vtable.async: allocates a Future closure (context+result trailing), runs inline if OOM or pool saturated (returns null); concurrent: error.ConcurrencyUnavailable when limit hit. Cancel protocol: per-thread packed Status{cancelation: 3-bit enum(none/parked/blocked/blocked_canceling/canceling/canceled/...), awaitable: AwaitableId(top ptr bits)}; blocking syscalls wrap in `Syscall.start()/checkCancel()/finish()` state machine; canceler CASes target status and sends SIG.IO (handler installed in init, no-op fn) causing EINTR; syscall loop retries on EINTR after checkCancel. Sleep = clock_nanosleep loop with same protocol. netRead = readv; netWrite = writev with header+data+splat vectorization.

### Evented/Uring/Kqueue/Dispatch (evented impls)
`Io.Evented = if (fiber.supported) switch (os) { .linux => Uring, bsd => Kqueue, darwin => Dispatch, else => void }`. fiber.zig: raw stackful coroutine support only for aarch64/riscv64/x86_64 — `Context{sp,fp,pc}` + `contextSwitch(*const Switch) *const Switch` inline asm clobbering all regs. Uring.zig (6.1k lines, WIP): per-thread IoUring rings (IORING_SETUP_COOP_TASKRUN|SINGLE_ISSUER), work-stealing M:N fibers (Fiber struct with context, cancel_status packed(u32){requested, awaiting: enum fd/group/nothing}), min_stack_size 60MB virtual per fiber, InitOptions{thread_limit, sync_limit, log2_ring_entries}. CRITICAL: Uring's vtable has netListenIp/netAccept/netConnectIp/netRead/netWrite/netLookup = *Unavailable stubs — evented networking NOT implemented in 0.16 (issue 31723). Kqueue/Dispatch are proofs of concept. So a production HTTP SDK today runs on Threaded; code written against Io gets evented for free later.

### Io.Reader interface (buffer-in-interface)
Io.Reader is a non-generic struct: `vtable: *const VTable, buffer: []u8, seek: usize, end: usize` — the buffer LIVES IN the interface; buffer[seek..end] is buffered data. VTable: `stream: *const fn(r: *Reader, w: *Writer, limit: Limit) StreamError!usize` (required; writes from logical position into a Writer; may instead fill r.buffer directly adjusting seek/end); optional `discard(r, limit) Error!usize`, `readVec(r, data: [][]u8) Error!usize`, `rebase(r, capacity) RebaseError!void` with defaults. Errors: StreamError={ReadFailed, WriteFailed, EndOfStream}; Error={ReadFailed, EndOfStream}; diagnostics live on the concrete impl (e.g. net.Stream.Reader.err: ?Error side-channel). High-level API operates on the buffer: peek(n)![]u8, take(n), toss(n), fill(n), takeByte, takeArray, takeSentinel, takeDelimiterExclusive/Inclusive (SSE line parsing!), streamDelimiter, streamExact/streamRemaining (pump to Writer), discardRemaining, readSliceAll, readAlloc, appendRemaining(gpa, list, limit), takeLeb128, takeStructPointer, peekStructPointer. `.fixed(buffer)` for in-memory; `.limited()`; Reader.Limited; Hashed wrapper. Composition = struct embedding: concrete impls hold `interface: Io.Reader` and recover self via @fieldParentPtr("interface", io_r).

### Io.Writer interface
Io.Writer: `vtable: *const VTable, buffer: []u8, end: usize`. VTable: required `drain: *const fn(w: *Writer, data: []const []const u8, splat: usize) Error!usize` (consumes w.buffer[0..end] first, then data slices; last data elem repeated splat times; returns bytes consumed from data only); optional `sendFile(w, file_reader: *File.Reader, limit) FileError!usize` (default unimplementedSendFile → error.Unimplemented = caller does own reads), `flush` (default drains until end==0), `rebase(w, preserve, capacity)`. Error = error{WriteFailed} only; diagnostics on concrete impl (.err field). API: write, writeAll, print(fmt,args), writeByte, splatByteAll, writeVec/writeVecAll, writableSlice/writableSliceGreedy+advance (zero-copy in-place encode), sendFileAll, printInt, writeUleb128, flush. Instances: `.fixed(buf)`, `Writer.Allocating` (allocator-backed growable, flush=no-op, fromArrayList/toArrayList), `Writer.Discarding`, `.failing`. Zero-length buffer = unbuffered is legal for drain-based writers.

### Io.net sockets and streams
net.IpAddress = union(enum){ip4: Ip4Address{bytes:[4]u8, port:u16}, ip6: Ip6Address{port, bytes:[16]u8, flow, interface}} with parse/parseLiteral (pure) and resolve(io,...) (scope ids need ioctl). Ops as methods: `address.listen(io, ListenOptions{kernel_backlog=128, reuse_address, mode=.stream, protocol=.tcp}) ListenError!Server`, `address.bind(io, BindOptions) !Socket` (UDP), `address.connect(io, ConnectOptions{mode, protocol, timeout: Io.Timeout=.none}) ConnectError!Stream`. Server.accept(io) AcceptError!Stream. Socket{handle: posix.fd_t, address} with send/sendMany/receive/receiveTimeout/receiveManyTimeout (receive goes through io.operate(.net_receive)). net.Stream{socket} = connection-oriented: `stream.reader(io, buffer) Stream.Reader` and `stream.writer(io, buffer) Stream.Writer` — these adapt to Io.Reader/Io.Writer: Stream.Reader{io, interface: Io.Reader, stream, err: ?Error} where readVec impl calls io.vtable.netRead with iovecs from Io.Reader.writableVector, 0 bytes ⇒ error.EndOfStream, real error stored in .err and error.ReadFailed returned. Stream.Writer.drain calls io.vtable.netWrite(handle, buffered, data, splat). close/shutdown via vtable. UnixAddress.listen/connect for UDS.

### DNS + happy-eyeballs connect (idiomatic async example)
HostName.lookup(host, io, resolved: *Io.Queue(LookupResult), .{.port, .canonical_name_buffer, .family}) LookupError!void — vtable netLookup pushes results into the queue and closes it. HostName.connect (HostName.zig:274) is THE idiomatic pattern: `var q: Io.Queue(ConnectError!Stream) = .init(&buf32); var cm = io.async(connectMany, .{host, io, port, &q, options}); defer { _ = cm.cancel(io) catch {}; while (q.getOneUncancelable(io)) |loser| { if (loser) |s| s.close(io) else |_| {} } else |err| switch(err){error.Closed=>{}} }` then `while (q.getOne(io)) |result| {...} else |err| switch (err) { error.Closed => return lookup error via cm.await(io), error.Canceled => |e| return e }`. connectMany itself: spawns lookup future via io.async, then `var group: Io.Group = .init; defer group.cancel(io);` and per-address `group.async(io, enqueueConnection, .{addr, io, results, options})`; first winner returned, all losers canceled/closed. Works even single-threaded (sequential fallback).

### Time, sleep, timeout, deadline
Io.Timestamp{nanoseconds: i96}; Io.Duration{nanoseconds: i96} with fromMilliseconds/fromSeconds etc; Clock = enum{real, awake, boot, cpu_process, cpu_thread} with `now(clock, io) Timestamp` (not cancelable) and `resolution(clock, io)`; Clock.Timestamp/Clock.Duration pair raw value with clock for type safety. `Io.Timeout = union(enum){none, duration: Clock.Duration, deadline: Clock.Timestamp}` with Error=error{Timeout}, toDeadline(io), sleep(io). Sleeping: `io.sleep(duration: Duration, clock: Clock) Cancelable!void`, `timeout.sleep(io)`, `clock_timestamp.wait(io)`. Timeouts on operations: ConnectOptions.timeout; Socket.receiveTimeout; `io.operateTimeout(operation, timeout) OperateTimeoutError!Operation.Result` gives a timeout to any Operation (currently FileReadStreaming, FileWriteStreaming, DeviceIoControl, NetReceive) via Batch.awaitConcurrent(io, timeout); Event.waitTimeout(io, timeout); futexWaitTimeout. General deadline pattern for arbitrary work: spawn via async and race, or use operateTimeout for supported ops.

### Batch/Operation (low-level concurrency)
Io.Batch: caller provides `storage: []Operation.Storage`; `init(storage)`, `add(op) u32`/`addAt(index, op)`, `awaitAsync(io) Cancelable!void` (waits ≥1 completion, concurrency optional), `awaitConcurrent(io, timeout) (ConcurrentError||Cancelable||Timeout.Error)!void` (concurrency required), `next() ?Completion{index, result}` iterator, `cancel(io)`. Operation = union(enum){file_read_streaming, file_write_streaming, device_io_control, net_receive} with typed per-op Result via comptime-built union. This is the zero-allocation path; Future/Group allocate task memory.

### Sync primitives (Io-integrated)
All under Io, all take io param: Io.Mutex extern struct{state: atomic(enum(u32){unlocked, locked_once, contended})} with tryLock() bool, lock(io) Cancelable!void, lockUncancelable(io), unlock(io) — futex-based via io.futexWait/futexWake. Io.Condition{state packed(u32){waiters:u16, signals:u16}, epoch} with wait(io, mutex) Cancelable!void, waitUncancelable, signal(io), broadcast(io). Io.Event enum(u32){unset, waiting, is_set} with wait/waitTimeout/set/reset/isSet. Io.RwLock{state, mutex, semaphore} lock/lockShared + uncancelable variants. Io.Semaphore{mutex, cond, permits} wait/post. Io.futexWait(io, comptime T, ptr, expected) where sizeOf(T)==4. Pattern: every blocking primitive has cancelable (error.Canceled) and Uncancelable variants; unlock paths use Uncancelable to avoid errdefer complexity.

### File I/O and File.Reader/Writer adapters
Io.File{handle: posix.fd_t, flags: {nonblocking: bool}}; stdout()/stderr()/stdin() constructors. Methods take io: readStreaming(io, buffers), writeStreamingAll(io, bytes), readPositional, close(io), stat(io), etc. Io.Dir{...} with .cwd(), openFile(dir, io, sub_path, options) File.OpenError!File. File.Reader (File/Reader.zig) memoizes size/seek pos/mode (positional vs streaming vs *_simple fallbacks) and holds `interface: Io.Reader` + `err: ?Error` + `io: Io` + `file: File`; init(file, io, buffer), initStreaming; seekTo/seekBy with read-fallback; logicalPos = pos - bufferedLen. File.Writer mirrors it (mode .streaming). This concrete-struct-wrapping-interface + err-side-channel pattern is the template every SDK transport should follow.

### How libraries accept Io; Juicy Main; testing
Convention (relnotes + std.http.Client): main() constructs the Io; libraries accept `io: Io` parameter by value or store it as a plain field on a context struct (std.http.Client stores `allocator: Allocator, io: Io`). Io is 16 bytes on 64-bit, trivially copyable, no lifetime beyond its backing implementation struct. Fallback when stranded: `var threaded: Io.Threaded = .init_single_threaded; const io = threaded.io();` (discouraged, like page_allocator). `pub fn main(init: std.process.Init)` gives init.io (default impl per target), init.gpa, init.arena: *ArenaAllocator, init.minimal.args, init.environ_map, init.preopens. Tests use `std.testing.io` (= testing.io_instance.io(), an Io.Threaded). Entropy: io.random(buffer) void (CSPRNG), io.randomSecure(buffer) RandomSecureError!void. Env vars are non-global now — must be passed in.

### std.http.Client as reference HTTP+TLS consumer
Client{allocator, io, ca_bundle_lock: Io.RwLock, ca_bundle, tls_buffer_size, pool}. Connection allocated as single block embedding host bytes + all buffers; Plain: stream_writer = stream.writer(io, socket_write_buffer), stream_reader = stream.reader(io, socket_read_buffer). TLS layering: std.crypto.tls.Client.init(&conn.stream_reader.interface, &conn.stream_writer.interface, .{.host, .ca, .read_buffer, .write_buffer, .entropy (from io.random), .realtime_now, ...}) — TLS client itself is Io-agnostic, purely Reader→Reader/Writer→Writer composition; error.ReadFailed/WriteFailed unwrapped via stream_reader.err.?/stream_writer.err.?. Request flow: client.request(method, uri, options) → req.sendBodiless()/sendBody(buffer) → req.receiveHead(redirect_buffer) ReceiveHeadError!Response → response.reader(transfer_buffer) *Io.Reader (or readerDecompressing) for streaming body. TLS min buffer: std.crypto.tls.Client.min_buffer_len. `std.options.http_disable_tls`, `std.options.networking` gate features.

### Io.Limit and vectorized write shape
Io.Limit = enum(usize){nothing=0, unlimited=maxInt, _} with .limited(n), .limited64, .min/.max, .slice(s), .subtract, .toInt() ?usize — used pervasively for read/write caps. The canonical write shape through the whole stack (Writer.drain, netWrite vtable, fileWritePositional) is `(header: []const u8, data: []const []const u8, splat: usize)` — header/buffered bytes first, then iovec-style slices with the last slice logically repeated splat times; implementations map this to writev with max_iovecs_len=8 and a 64-byte splat_buffer. Reader.writableVector/writableVectorPosix bridge Reader buffers to iovecs.

## Port implications
Design the entire ai.zig SDK around passing/storing `Io` by value (16 bytes: userdata + vtable ptr), exactly like std.http.Client: `pub const Client = struct { allocator: Allocator, io: Io, ... }`. Never construct Io inside the library; tests use std.testing.io. Every public op that blocks must carry `Io.Cancelable` (error.Canceled) in its error set — this is free if you only compose Io primitives. Streaming: model streamText's part stream as `Io.Queue(StreamPart)` with a caller-provided buffer; producer task spawned via `io.async` (portable, degrades to sequential single-threaded) writes parts and `close()`s the queue; consumer loop is `while (q.getOne(io)) |part| ... else |err| switch (err) { error.Closed => done, error.Canceled => propagate }`. Abort/AbortSignal maps 1:1 to Future.cancel/Group.cancel — use the `var fut = io.async(...); defer _ = fut.cancel(io) catch {};` pattern; parallel tool execution and provider racing map to Io.Group / Io.Select. SSE parsing should be a pure `*Io.Reader → events` layer using takeDelimiterExclusive/peekDelimiterInclusive, mirroring how tls.Client wraps Reader/Writer without touching Io — keeps it testable with Reader.fixed. HTTP/TLS come free from std.http.Client (request → sendBody → receiveHead → response.reader). Timeouts: Io.Timeout union everywhere (ConnectOptions.timeout, operateTimeout), not ad-hoc ns integers; expose Clock.Duration in options structs. Hard parts: (1) Future(T)/async are comptime-generic — the C ABI layer must instead call vtable.async directly with type-erased context/result buffers (the vtable signature is already fully type-erased, so a C wrapper is natural: export the start-callback trampoline); (2) error.Canceled is one-shot — FFI callbacks that swallow it must call io.recancel(); (3) Evented/Uring has no networking yet, so ship on Io.Threaded and document that concurrency limits come from InitOptions{async_limit, concurrent_limit}; (4) Reader/Writer buffers are always caller-provided — surface buffer-size knobs in the C API like http.Client's read_buffer_size/tls_buffer_size; Writer.Allocating covers the convenience path for Python/Rust wrappers.
