# zig-http-json

## Summary
Zig 0.16 std.http.Client is fully integrated with the new std.Io interface: `http.Client` is a plain struct `{ .allocator, .io }` (no init fn) holding a thread-safe LRU ConnectionPool, TLS state, and proxy pointers. The request lifecycle is request() -> sendBodiless()/sendBody()/sendBodyComplete() -> receiveHead(redirect_buffer) -> response.reader(transfer_buffer), where every stream is a non-generic `*std.Io.Reader`/`*std.Io.Writer` vtable interface. Chunked transfer encoding is de-framed transparently by http.Reader (state machine in std/http.zig with ChunkParser), and request bodies stream via http.BodyWriter where each flush of a chunked body emits one wire chunk. Redirects are followed automatically for bodiless requests (RedirectBehavior enum with count), gzip/deflate/zstd decompression is built in via readerDecompressing, TLS 1.2/1.3 is native (std.crypto.tls.Client, wraps reader/writer pair, ca bundle rescanned on first HTTPS request). std.json has three layers: streaming Scanner (push-buffer, BufferUnderrun protocol, partial tokens), json.Reader (pulls from *Io.Reader), and typed parseFromSlice/parseFromTokenSource into arena-backed Parsed(T); dynamic Value; writer-based Stringify with beginObject/objectField/write plus one-shot valueAlloc. I built and ran a complete worked example on 0.16.0: POST JSON with content-length body, receive a chunked text/event-stream response, parse SSE lines incrementally with takeDelimiter and json.parseFromSlice per event — verified events arrive incrementally (t=0/311/622ms) before the response ends. Gaps: no SSE parser in stdlib, connectUnix is broken dead code, no per-request timeout, no HTTP/2, no client-side WebSocket, takeDelimiterExclusive does not consume the delimiter (use takeDelimiter), and std.time.milliTimestamp/Thread.sleep/ResetEvent are gone (time/sleep go through Io).

## Key findings
### http.Client struct shape and Io integration
Client is field-initialized, no init(): `var client: std.http.Client = .{ .allocator = gpa, .io = io };` then `client.deinit()`. Fields: allocator (must be thread-safe), io: std.Io, ca_bundle + ca_bundle_lock (Io.RwLock), tls_buffer_size (default std.crypto.tls.Client.min_buffer_len = max_ciphertext_record_len ~16.6KB), ssl_key_log, now: ?Io.Timestamp (cert-expiry clock; first HTTPS request rescans system root certs -> error.CertificateBundleLoadFailure on failure), connection_pool (LRU, free_size=32), read_buffer_size=8192 (must hold entire response head or error.HttpHeadersOversize), write_buffer_size=1024, http_proxy/https_proxy: ?*Proxy (externally owned). TLS can be compiled out via std.options.http_disable_tls.

### Request lifecycle signatures
`pub fn request(client: *Client, method: http.Method, uri: std.Uri, options: RequestOptions) RequestError!Request` — connects (or reuses pooled connection), does NOT send anything. RequestOptions: {version=.@"HTTP/1.1", handle_continue=true, keep_alive=true, redirect_behavior: RedirectBehavior = @enumFromInt(3), connection: ?*Connection=null, headers: Request.Headers, extra_headers: []const http.Header = &.{}, privileged_headers}. Then set `req.transfer_encoding = .{ .content_length = n }` or `.chunked` (default .none). Send: `req.sendBodiless()` (asserts no body); `var bw = try req.sendBody(buffer) -> http.BodyWriter` then `bw.writer.writeAll(...)` + `try bw.end()`; or `req.sendBodyComplete(body: []u8)` — NOTE takes MUTABLE []u8 (uses it as the writer buffer). Receive: `var response = try req.receiveHead(redirect_buffer: []u8) ReceiveHeadError!Response` — blocks until full head; follows redirects internally (loop), needs redirect_buffer to survive URI merging (8KB recommended, &.{} if redirect_behavior=.unhandled); returns error.RedirectRequiresResend if body already sent. `req.deinit()` returns connection to pool (discards unread body to enable keep-alive reuse). Standard headers overridable via Headers.Value union {default, omit, override: []const u8} for host/authorization/user_agent/connection/accept_encoding/content_type; arbitrary headers via extra_headers (externally-owned, must outlive request, kept across redirects; privileged_headers stripped on cross-domain redirect).

### Response and body streaming (chunked de-framing)
Response = { request: *Request, head: Head }. Head fields: version, status: http.Status (enum(u10) with .class() -> informational/success/redirect/client_error/server_error and .phrase()), reason, location, content_type, content_disposition, keep_alive, content_length: ?u64, transfer_encoding: http.TransferEncoding (enum{chunked,none}), content_encoding (enum{zstd,gzip,deflate,compress,identity}); head.iterateHeaders() -> http.HeaderIterator for all raw headers. `response.reader(transfer_buffer: []u8) *std.Io.Reader` — call once; INVALIDATES head strings (invalidateStrings). Internally http.Reader.bodyReader: for .chunked installs vtable {stream=chunkedStream, discard=chunkedDiscard} using ChunkParser to strip chunk framing (trailers parsed at 0-chunk into reader.trailers, exposed via response.iterateTrailers()); for content-length installs contentLengthStream tracking remaining; for .none w/o content-length returns connection reader directly (EOF-terminated). transfer_buffer is the returned Reader's buffer — line-based reads (takeDelimiter) are capped by its capacity (error.StreamTooLong). `response.readerDecompressing(transfer_buffer, decompress: *http.Decompress, decompress_buffer)` layers flate/zstd. On error.ReadFailed, `response.bodyErr() ?http.Reader.BodyError` gives {HttpChunkInvalid, HttpChunkTruncated, HttpHeadersOversize}. Default accept-encoding advertises gzip+deflate (zstd exists but off by default).

### Streaming request body via http.BodyWriter
BodyWriter = { http_protocol_output: *Writer, state: State union{none, content_length: u64, chunk_len: usize, end}, writer: std.Io.Writer }. Client fills vtable per transfer encoding: chunkedDrain/contentLengthDrain/noneDrain (+SendFile variants incl. sendfile lowering). With chunked + empty buffer (`sendBody(&.{})`), every writer.writeAll becomes one wire chunk (drain writes hex `%08x\r\n` header template + payload + \r\n); with a buffer, flush() emits buffered bytes as one chunk. `end()` asserts content_length hit 0 or writes `0\r\n\r\n` terminator; `endChunked(.{ .trailers = &.{...} })` sends HTTP trailers. content_length state counts down and asserts exact match. Server-side BodyWriter has eliding vtable for HEAD (isEliding). Verified pattern from stdlib tests: `req.transfer_encoding = .chunked; var bw = try req.sendBody(&.{}); var resp = try req.receiveHead(&buf); try bw.writer.writeAll("..."); try bw.end();` (receiveHead can be called before finishing the body for full duplex).

### Redirects, proxy, connection pool internals
RedirectBehavior = enum(u16) { not_allowed=0, unhandled=maxInt(u16), _ } — .init(n) for n redirects; receiveHead auto-follows for bodiless requests: skips redirect body, resolves Location via uri.resolveInPlace into caller's redirect_buffer, changes POST->GET on 303/301/302, strips privileged_headers cross-domain, reconnects. ConnectionPool: Io.Mutex-guarded DoublyLinkedLists (used/free), findConnection matches host (case-insensitive HostName.eql)/port/protocol, release() destroys if closing or pool full, proxied connections prepended (deprioritized). Connection is one allocation holding [Tls struct|Plain struct] + host bytes + all read/write buffers (Plain: read_buffer_size + write_buffer_size; Tls adds tls_buffer_size*3 + read_buffer_size). Proxy = {protocol, host: HostName, authorization: ?[]const u8 (pre-encoded Basic), port, supports_connect}; client.initDefaultProxies(arena, environ_map) reads http_proxy/HTTP_PROXY/all_proxy env vars; HTTPS through proxy uses CONNECT tunneling (connectProxied), fallback to plain forward-proxy mode with absolute-form request line. connectUnix exists but is BROKEN dead code (references nonexistent ConnectionPool.Node fields; only compiles because unreferenced) — unix-socket providers (e.g. local runtimes) need a custom path.

### TLS support and limits
std.crypto.tls.Client: TLS 1.2 + 1.3 only. `init(input: *Io.Reader, output: *Io.Writer, options: Options) InitError!Client` — performs handshake inline; input buffer capacity must be >= min_buffer_len = tls.max_ciphertext_record_len (2^14+256+5). Options: host: union{no_verification, explicit: []const u8}, ca: union{no_verification, self_signed, bundle:{gpa, io, lock: *Io.RwLock, bundle: *Certificate.Bundle}}, write_buffer/read_buffer: []u8, entropy: *const [240]u8, realtime_now: Io.Timestamp, ssl_key_log: ?*SslKeyLog (SSLKEYLOGFILE-style), allow_truncation_attacks: bool=false (http.Client sets true since Content-Length detects truncation), alert out-param. Exposes .reader/.writer (plaintext) over .input/.output (ciphertext). ReadError includes TlsAlert/TlsBadRecordMac/etc; http Connection.getReadError unwraps. No session resumption/ALPN/HTTP2; client cert auth not exposed via http.Client.

### Verified worked example: POST JSON + stream SSE (runs on 0.16.0)
Compiled and executed; events verified incremental (t=0/311/622ms). ### Io.Reader line primitives + delimiter gotcha (critical for SSE)
takeDelimiterExclusive(d) advances up to but NOT past the delimiter -> naive loop spins/misbehaves (empirically broke my first demo). Correct SSE line iterator: `takeDelimiter(d) error{ReadFailed,StreamTooLong}!?[]u8` — consumes delimiter, treats EOF-as-delimiter, returns null at clean EOF. Other primitives on *Io.Reader: peekGreedy(n)/toss(n)/buffered() for chunk-at-a-time (used by json.Reader.refillBuffer), fillMore(), stream(w, limit), streamRemaining(w), discardRemaining(), allocRemaining(gpa, limit), appendRemaining(gpa, list, limit), readSliceAll, takeByte, takeDelimiterInclusive (includes delimiter, error.EndOfStream at EOF), peekDelimiterExclusive. All delimiter fns return error.StreamTooLong if no delimiter within buffer capacity, so the transfer_buffer passed to response.reader() bounds max SSE line length. For unbounded lines, accumulate via streaming into Writer.Allocating instead.

### One-shot Client.fetch
`client.fetch(options: FetchOptions) FetchError!FetchResult` where FetchOptions = { redirect_buffer: ?[]u8 (null=heap 8KB), decompress_buffer: ?[]u8, redirect_behavior: ?RedirectBehavior, response_writer: ?*Io.Writer (null = body discarded), location: union{url: []const u8, uri: Uri}, method: ?Method (default POST if payload else GET), payload: ?[]const u8, raw_uri, keep_alive=true, headers/extra_headers/privileged_headers } and FetchResult = { status: http.Status } only. Payload sent with content-length; automatic decompression (zstd window or flate max_window heap-allocated if needed). Good template for the port's non-streaming generateText path but discards response headers — need the manual request() flow to read them.

### std.json.Scanner (streaming, push model)
Scanner = struct { state, stack: BitStack (O(depth/8) mem, only allocation), input: []const u8, cursor, is_end_of_input, diagnostics: ?*Diagnostics }. initStreaming(gpa) / initCompleteInput(gpa, slice); feedInput(slice) after error.BufferUnderrun from next(); endInput() at stream end. `next() NextError!Token` where NextError = error{SyntaxError, UnexpectedEndOfInput} || Allocator.Error || error{BufferUnderrun}. Token = union(enum){ object_begin/object_end/array_begin/array_end/true/false/null, number/partial_number: []const u8, allocated_number: []u8, string/partial_string: []const u8, partial_string_escaped_1..4: [n]u8, allocated_string: []u8, end_of_document } — partial_* tokens carry value fragments split across buffer boundaries/escapes; slices point into current input buffer (invalidated by next feedInput). nextAlloc(gpa, .alloc_if_needed|.alloc_always) concatenates partials (max default_max_value_len=4MiB). skipValue()/skipUntilStackHeight(h) for skipping; peekNextTokenType(); Diagnostics gives getLine()/getColumn()/getByteOffset() via enableDiagnostics. This push/BufferUnderrun design is ideal for the AI SDK's incremental partial-JSON streaming (streamObject): feed each SSE delta, read tokens until underrun.

### std.json.Reader (pull from *Io.Reader) and typed parsing
json.Reader = { scanner, reader: *std.Io.Reader }; .init(gpa, io_reader); same next()/nextAlloc()/skipValue() API but auto-refills via reader.peekGreedy(1)+toss (NextError = std.Io.Reader.Error || json.Error || Allocator.Error). Typed: `parseFromSlice(comptime T, gpa, s, options) ParseError(Scanner)!Parsed(T)` and `parseFromTokenSource(T, gpa, scanner_or_reader: anytype, options)` — Parsed(T) = { arena: *ArenaAllocator, value: T }, deinit() frees everything (arena model; Leaky variants for caller-owned arenas). ParseOptions: { duplicate_field_behavior: enum{use_first,@"error",use_last} = .@"error", ignore_unknown_fields: bool = false, max_value_len, allocate: ?AllocWhen, parse_numbers: bool = true }. Supports structs (default field values = optional fields), tagged unions ({"tag": value} shape), enums, optionals, slices, tuples; custom decode hook via `pub fn jsonParse(gpa, source: anytype, options) !T` + innerParse re-entry; jsonParseFromValue for Value-stage. ParseFromValueError adds UnexpectedToken/InvalidNumber/Overflow/InvalidEnumTag/DuplicateField/UnknownField/MissingField/LengthMismatch. parseFromValue(T, gpa, value: Value, options) converts dynamic->typed.

### std.json.Value (dynamic) and Stringify
Value = union(enum){ null, bool, integer: i64, float: f64, number_string: []const u8 (overflow/precision fallback), string, array: std.array_list.Managed(Value), object: ObjectMap (StringArrayHashMap) } — parse via parseFromSlice(Value, ...); non-recursive parser (explicit stack). Stringify (std/json/Stringify.zig): stateful writer `var s: Stringify = .{ .writer = *Io.Writer, .options = .{} }` with beginObject/objectField(key)/objectFieldRaw/write(anytype)/print(fmt,args)/beginWriteRaw+endWriteRaw (stream long strings raw)/beginArray/endArray/endObject; grammar asserted in Debug up to depth 256. Options: { whitespace: enum{minified,indent_1..indent_8,indent_tab}=.minified, emit_null_optional_fields=true, emit_strings_as_arrays=false, escape_unicode=false, emit_nonportable_numbers_as_strings=false (ints beyond ±2^53 as strings) }. One-shots: `Stringify.value(v, options, writer)`, `Stringify.valueAlloc(gpa, v, options) ![]u8`; `std.json.fmt(v, options)` adapter for {f} format specifier. Custom encode hook: `pub fn jsonStringify(self, jw: anytype) !void`. Floats emitted as f64 when exactly representable else quoted string.

### http.Server for UI-message-stream endpoint
`http.Server.init(in: *Io.Reader, out: *Io.Writer)` over an accepted Io.net.Stream's reader/writer; `s.receiveHead() ReceiveHeadError!Request`; request.head has method/target/version/content_type etc.; request body via `request.readerExpectContinue(buffer) !*Io.Reader` (handles Expect: 100-continue). Respond: `request.respond(content, RespondOptions)` one-shot, or `var response: http.BodyWriter = try request.respondStreaming(buffer, .{ .content_length = null (null => transfer-encoding: chunked), .respond_options = .{ .status, .extra_headers, .transfer_encoding, .keep_alive } })` then writer.writeAll + response.flush() per SSE event (verified: each flush = one chunk) + response.end() or endChunked(.{ .trailers }). HEAD handled by eliding writer. Server-side WebSocket upgrade included: request.upgradeRequested() -> respondWebSocket(.{}) -> WebSocket with readSmallMessage/writeMessage (RFC6455, small frames only). No routing/listener loop — bring your own accept loop via Io.net.Server.

### Io instance creation and concurrency model
Io = { userdata: ?*anyopaque, vtable: *const VTable } fat pointer — trivially carryable through C FFI. Production: `var threaded: Io.Threaded = .init(gpa, .{}); const io = threaded.io();` (installs SIGIO/SIGPIPE handlers for cancellation; deinit joins workers). Single-threaded fallback: `Io.Threaded.init_single_threaded` (concurrent() fails with ConcurrencyUnavailable, cancellation no-ops). Tests: std.testing.io. New main: `pub fn main(init: std.process.Init)` provides init.io/init.gpa/init.arena ("Juicy Main"). Primitives: std.Io.async(io, f, args)->Future (WARNING: may run inline when no worker available — my io.async server demo deadlocked; stdlib http tests use std.Thread.spawn for the server side), io.concurrent (guaranteed parallel or error), Io.Group, Io.Queue(T) (MPMC suspend/resume channel — natural backing for stream-part pipes), Io.Select, cancellation via future.cancel(io), Io.Mutex/Condition/RwLock/ResetEvent (std.Thread.ResetEvent is GONE). Time: Io.Clock.real/.awake .now(io) -> Timestamp (.toMilliseconds()), io.sleep(Duration.fromMilliseconds(n), clock); std.time.milliTimestamp/std.Thread.sleep removed. Alternative event-loop Io impls exist but incomplete: Io.Uring (no networking yet), Io.Kqueue, Io.Dispatch — Threaded is the only production-ready one in 0.16.

### Networking layer under the client
Io.net: HostName (validated, .max_len, .bytes; `host.connect(io, port, .{ .mode = .stream }) !Stream` does parallel-DNS + happy-eyeballs style connect w/ cancellation per release notes), IpAddress.listen(io, .{ .reuse_address }) -> Server, server.accept(io) -> Stream, stream.reader(io, buffer) -> Stream.Reader (.interface is *Io.Reader) / stream.writer(io, buffer) -> Stream.Writer, stream.close(io). ConnectTcpOptions has `timeout: Io.Timeout = .none` (connect-phase only). No per-request/read deadline API on http.Client — request timeouts must be built from cancellation (async task + future.cancel) or a custom Io wrapper; this is a port-level responsibility for AI SDK's abortSignal equivalent.

### Gaps to fill for ai.zig
(1) SSE/EventSource parser: absent from stdlib (grep confirms zero server-sent-events code) — must implement per WHATWG spec (data:/event:/id:/retry:, comment lines, multi-line data joining, \r\n|\r|\n line endings, BOM) over *Io.Reader; bound by transfer_buffer or use growable accumulation. (2) Client-side WebSocket: none (server-side only). (3) HTTP/2: none. (4) connectUnix broken. (5) Per-request timeout/deadline: none. (6) Request retry w/ backoff: none. (7) zstd not in default accept-encoding (set req.accept_encoding[@intFromEnum(.zstd)] = true if desired). (8) Multipart/form-data encoding: none in stdlib. (9) Client does not expose response header allocation — head strings are views into connection buffer invalidated by response.reader(); copy needed before streaming (relevant for AI SDK's response.headers surface). (10) Incremental/partial JSON value assembly (AI SDK streamObject's partial object views) not provided — Scanner gives tokens; partial-object materialization is port work. (11) The cached 0.16 stdlib's http/test.zig is empty; the full test suite lives in the ziglang corpus at lib/std/http/test.zig (identical Client.zig) — best reference for canonical usage.

## Port implications
The HTTP layer needs almost no abstraction invention: http.Client is already an io-parameterized, thread-safe, pooled client whose every stream is a non-generic *Io.Reader/*Io.Writer vtable object — the port's provider HTTP layer can be a thin struct holding {allocator, io, *http.Client, base_url, headers}. AI SDK's fetch-replacement hook maps naturally to a vtable/function-pointer taking (method, uri, headers, body_reader) -> Response. Streaming is the natural default: chunked/content-length de-framing, gzip/deflate/zstd, redirects, proxies, and TLS 1.2/1.3 are stdlib-provided; the verified POST-JSON + SSE-parse loop (scratchpad/sse_demo.zig) is the exact skeleton for streamText's provider call. Port work concentrates in: (a) an SSE decoder over *Io.Reader (whatwg-compliant; beware takeDelimiterExclusive vs takeDelimiter semantics and StreamTooLong = line exceeds caller buffer); (b) partial-JSON assembly for streamObject on top of json.Scanner's push/BufferUnderrun model (feed SSE deltas, emit tokens incl. partial_string fragments); (c) timeouts/abort: no per-request deadline exists — implement AbortSignal as Io cancellation (io.async future.cancel) and connect-phase Io.Timeout; (d) retry/backoff via io.sleep. Stream plumbing: Io.Queue(T) is the ReadableStream analogue for fan-out between the SSE decoder task and consumer; beware io.async may execute inline when concurrency is unavailable (deadlock risk observed) — use io.concurrent or dedicated threads for pump loops. For C FFI: Io is a {userdata, vtable} fat pointer, so the C layer can own a Threaded instance and hand out opaque stream handles wrapping *Io.Reader; all buffers (redirect, transfer, TLS) are caller-provided with strict lifetime rules (response head strings are invalidated by response.reader()) — copy-at-boundary semantics are mandatory for Python/Rust wrappers. Typed JSON decode of provider responses maps to parseFromSlice(T, ...) with .ignore_unknown_fields=true into arena-backed Parsed(T); serialization via Stringify.valueAlloc or streaming Stringify against the request BodyWriter.
