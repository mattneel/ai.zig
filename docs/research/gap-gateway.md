# gap:Gateway package internals (auth, metadata, model routing)

## Summary
The @ai-sdk/gateway package is a thin "identity provider": it serializes the normalized AI SDK v4 call options directly onto the wire and streams back normalized LanguageModelV4StreamPart SSE events, with no provider-specific prompt conversion. createGateway(options) returns a callable GatewayProvider (function + methods) covering 8 modalities (language, embedding, image, video, reranking, speech, transcription, realtime-over-WebSocket) plus metadata endpoints (getAvailableModels, getCredits, getSpendReport, getGenerationInfo) and 3 provider-executed web-search tools. Default baseURL is https://ai-gateway.vercel.sh/v4/ai. Auth is resolved per-request via an async headers resolvable: apiKey option or AI_GATEWAY_API_KEY env wins ('api-key' method); otherwise @vercel/oidc's getVercelOidcToken() ('oidc' method, backed by VERCEL_OIDC_TOKEN / Vercel request context — package is external, not vendored). Auth method is echoed in the 'ai-gateway-auth-method' header and parsed back to build contextual error messages. Ten error classes derive from abstract GatewayError (marker-symbol instanceof, statusCode, isRetryable = 408/409/429/>=500, generationId), constructed from a wire error schema {error:{message,type,param,code},generationId} via a type-string switch. Metadata (GET {baseURL}/config) is cached in the provider closure with 5-minute stale-while-revalidate and single-flight semantics. The `ai` core package statically imports gateway in exactly five places: default-global-provider fallback (globalThis.AI_SDK_DEFAULT_PROVIDER ?? gateway) in resolve-model.ts, GatewayError.isRetryable in retry logic, GatewayAuthenticationError detection in wrapGatewayError (used by generateText/streamText/generateObject/streamObject), GatewayModelId as the default string-model-id type, and top-level re-exports of createGateway/gateway.

## Key findings
### createGateway / GatewayProviderSettings
createGateway(options: GatewayProviderSettings = {}): GatewayProvider. Settings: baseURL? (default 'https://ai-gateway.vercel.sh/v4/ai', trailing slash stripped), apiKey? (fallback env AI_GATEWAY_API_KEY via loadOptionalSetting), teamIdOrSlug?, headers? (Record<string,string>), fetch? (FetchFunction), metadataCacheRefreshMillis? (default 300_000), _internal.currentDate? for tests. GatewayProvider extends ProviderV4 and is itself callable: (modelId)=>LanguageModelV4 (throws if called with `new`). Methods: chat/languageModel, embedding/embeddingModel/textEmbeddingModel (deprecated alias), image/imageModel, video/videoModel, reranking/rerankingModel, speech/speechModel, transcription/transcriptionModel, experimental_realtime (factory fn + .getToken), getAvailableModels, getCredits, getSpendReport, getGenerationInfo, tools. provider.specificationVersion='v4'. Module exports singleton `export const gateway = createGateway()` at load time. Every modality model is constructed with {provider:'gateway', baseURL, headers:getHeaders(async), fetch, o11yHeaders} — realtime instead gets {provider:'gateway.realtime', baseURL, teamIdOrSlug, createClientSecret}.

### Auth selection (getGatewayAuthToken)
async getGatewayAuthToken(options): Promise<{token:string; authMethod:'api-key'|'oidc'}>. Order: (1) options.apiKey else env AI_GATEWAY_API_KEY -> authMethod 'api-key'; (2) else await getVercelOidcToken() from @vercel/oidc@3.2.0 -> 'oidc'. Any throw is wrapped as GatewayAuthenticationError.createContextualError({apiKeyProvided:false, oidcTokenProvided:false, statusCode:401, cause}). No token caching/refresh in the gateway package itself: headers is a Resolvable async fn re-invoked on every request (so OIDC token re-read per call); refresh logic, if any, lives inside external @vercel/oidc (not vendored in repo; reads VERCEL_OIDC_TOKEN env and Vercel request context globalThis[Symbol.for('@vercel/request-context')]). getVercelRequestId() returns getContext().headers?.['x-vercel-id'].

### Request header protocol
Auth headers built per request: Authorization: 'Bearer <token>'; 'ai-gateway-protocol-version': '0.0.1'; 'ai-gateway-auth-method': 'api-key'|'oidc' (constant GATEWAY_AUTH_METHOD_HEADER); optional 'x-vercel-ai-gateway-team': teamIdOrSlug (VERCEL_AI_GATEWAY_TEAM_HEADER); user headers; then withUserAgentSuffix('ai-sdk/gateway/<VERSION>'). O11y headers (created once per provider, request-id resolved per call): env VERCEL_DEPLOYMENT_ID->'ai-o11y-deployment-id', VERCEL_ENV->'ai-o11y-environment', VERCEL_REGION->'ai-o11y-region', VERCEL_PROJECT_ID->'ai-o11y-project-id', request context x-vercel-id->'ai-o11y-request-id'. parseAuthMethod(headers) validates headers['ai-gateway-auth-method'] against union 'api-key'|'oidc' (else undefined) — used when converting errors so the auth-failure message matches the method actually used.

### Error class inventory
Abstract GatewayError extends Error: fields statusCode (default 500), cause?, generationId? (appended to message as ' [<id>]'), isRetryable (default: statusCode===408||409||429||>=500); marker Symbol.for('vercel.ai.gateway.error') + per-class Symbol.for('vercel.ai.gateway.error.<Name>') for isInstance. Concrete classes (name / type string / default status): GatewayAuthenticationError/'authentication_error'/401 (has createContextualError with 3 message variants keyed on apiKeyProvided/oidcTokenProvided); GatewayInvalidRequestError/'invalid_request_error'/400; GatewayForbiddenError/'forbidden'/403 (+ruleId from param {ruleId:string}); GatewayModelNotFoundError/'model_not_found'/404 (+modelId from param {modelId:string}); GatewayTimeoutError/'timeout_error'/408 (client-side, from undici codes UND_ERR_HEADERS_TIMEOUT/UND_ERR_BODY_TIMEOUT/UND_ERR_CONNECT_TIMEOUT, incl. cause chain of APICallError); GatewayFailedDependencyError/'failed_dependency'/424; GatewayRateLimitError/'rate_limit_exceeded'/429; GatewayInternalServerError/'internal_server_error'/500 (also the fallback for unknown type strings); GatewayResponseError/'response_error'/502 (+response, validationError — used when the error body fails schema validation, still extracts raw generationId).

### Error conversion pipeline
asGatewayError(error, authMethod?): pass through if already GatewayError; undici timeout -> GatewayTimeoutError.createTimeoutError; APICallError -> createGatewayErrorFromResponse(extractApiCallResponse(err), statusCode ?? 500, authMethod) where extractApiCallResponse returns err.data ?? secureJsonParse(err.responseBody) ?? {}; anything else -> createGatewayErrorFromResponse({}, 500). Wire error schema: {error:{message:string, type?:string|null, param?:unknown, code?:string|number|null}, generationId?:string|null}; dispatch on error.type string. Every model class catches request errors and rethrows `await asGatewayError(error, await parseAuthMethod(resolvedHeaders ?? {}))`.

### Metadata endpoint + caching
GatewayFetchMetadata.getAvailableModels(): GET `${baseURL}/config` -> {models: GatewayLanguageModelEntry[]}. Entry wire schema: {id, name, description?null, pricing?{input:string, output:string, input_cache_read?->cachedInputTokens, input_cache_write?->cacheCreationInputTokens} (USD-per-token as strings), specification:{specificationVersion:'v4' literal, provider:string, modelId:string}, modelType?:string} — entries with unknown modelType are filtered out against KNOWN_MODEL_TYPES=['embedding','image','language','realtime','reranking','speech','transcription','video'] (null/absent kept). getCredits(): GET `${new URL(baseURL).origin}/v1/credits` -> {balance:string, total_used->totalUsed:string}. Caching lives in createGateway closure: pendingMetadata:Promise|null, metadataCache, lastFetchTime; on call, refetch iff !pendingMetadata || now-lastFetchTime > metadataCacheRefreshMillis (default 5 min); returns metadataCache immediately if present (stale-while-revalidate) else the in-flight promise (single-flight).

### GatewayLanguageModel wire protocol
implements LanguageModelV4; specificationVersion 'v4'; supportedUrls {'*/*':[/.*/]} (all URL-referenced files passed through — gateway fetches server-side). doGenerate/doStream POST `${baseURL}/language-model`. Request body = the ENTIRE LanguageModelV4CallOptions object minus abortSignal, JSON-serialized, after in-place base64 encoding of binary file data: for prompt message parts of type 'file'/'reasoning-file' and 'tool-result' outputs with output.type==='content' containing file parts, {type:'data', data:Uint8Array} -> base64 string. Extra headers: 'ai-language-model-specification-version':'4', 'ai-language-model-id':modelId, 'ai-language-model-streaming':'true'|'false'. doGenerate response body IS the LanguageModelV4GenerateResult (parsed with z.any(), spread; adds request:{body:args}, response:{headers, body:rawValue}, warnings:[]). doStream: SSE (createEventSourceResponseHandler(z.any())); each `data:` event is a LanguageModelV4StreamPart verbatim; TransformStream prepends {type:'stream-start',warnings} if any, drops 'raw' parts unless options.includeRawChunks, revives response-metadata.timestamp string->Date, and routes parse failures to controller.error. So unlike normal providers there is zero prompt/response mapping — the SDK's normalized types are the wire schema. Also exposes static WORKFLOW_SERIALIZE/WORKFLOW_DESERIALIZE hooks (serializeModelOptions) for workflow persistence.

### Other modality endpoints
All POST `${baseURL}/<kind>-model` with headers 'ai-<kind>-model-specification-version':'4' and 'ai-model-id':modelId (note: language model uniquely uses 'ai-language-model-id' + streaming header): embedding-model (body {values, providerOptions?}, resp {embeddings:number[][], usage?{tokens}, warnings?, providerMetadata?}; maxEmbeddingsPerCall=2048, supportsParallelCalls=true), image-model, video-model (SSE response, accept:text/event-stream, body {prompt,n,aspectRatio?,resolution?,duration?,fps?,seed?,generateAudio?,providerOptions?,image?,frameImages?,inputReferences?} with base64 file encoding), reranking-model, speech-model, transcription-model. Realtime: WebSocket at `${baseURL http->ws}/realtime-model?ai-model-id=<id>`; auth via Sec-WebSocket-Protocol subprotocols ['ai-gateway-realtime.v1', 'ai-gateway-auth.<token>', 'ai-gateway-team.<base64url(teamIdOrSlug)>'] (browser WS can't set headers); short-lived client secret (vcst_) minted server-side only via POST `${origin}/v1/realtime/client-secrets` body {model, expiresIn?} -> {token, expiresAt?:epoch-seconds}; throws if globalThis.window defined. Gateway realtime model is an identity codec (parseServerEvent/serializeClientEvent/buildSessionConfig all pass-through of normalized AI SDK realtime events).

### gateway-config.ts / gateway-provider-options.ts
GatewayConfig = {baseURL:string; headers?: Resolvable<Record<string,string|undefined>>; fetch?: FetchFunction} — headers is an async-resolvable so auth is re-derived per request. GatewayProviderOptions (typed catalog for providerOptions.gateway, validated server-side, has [key:string]:unknown index signature for forward-compat): byok?:Record<string,Array<Record<string,unknown>>>, caching?:'auto', disallowPromptTraining?:bool, has?:Array<'implicit-caching'>, hipaaCompliant?:bool, models?:string[] (fallbacks), only?:string[], order?:string[] (provider order), providerTimeouts?{byok?:Record<string,number>}, quotaEntityId?:string, serviceTier?:'flex'|'priority', sort?:'cost'|'tps'|'ttft', tags?:string[], user?:string, zeroDataRetention?:bool.

### gateway-tools / spend-report / generation-info
gatewayTools = {exaSearch, parallelSearch, perplexitySearch}: provider-executed tools (createProviderExecutedToolFactory) with ids 'gateway.exa_search'/'gateway.parallel_search'/'gateway.perplexity_search'; executed server-side by the gateway, client only supplies args schemas. GatewaySpendReport.getSpendReport(params): GET `${origin}/v1/report?start_date&end_date[&group_by∈day|user|model|tag|provider|credential_type][&date_part∈day|hour][&user_id][&model][&provider][&credential_type∈byok|system][&tags=csv]` -> {results:[{day?,hour?,user?,model?,tag?,provider?,credential_type?, total_cost, market_cost?, input_tokens?, output_tokens?, cached_input_tokens?, cache_creation_input_tokens?, reasoning_tokens?, request_count?}]} snake_case->camelCase. GatewayGenerationInfoFetcher.getGenerationInfo({id:'gen_<ulid>'}): GET `${origin}/v1/generation?id=<urlencoded>` -> {data:{id,total_cost,upstream_inference_cost,usage,created_at,model,is_byok,provider_name,streamed,finish_reason,latency,generation_time,native_tokens_prompt/completion/reasoning/cached/cache_creation,billable_web_search_calls}} unwrapped+camelCased (native_tokens_prompt->promptTokens etc.). Both resolve endpoints against URL origin, not the /v4/ai path.

### What the `ai` package calls on gateway
Exactly five static couplings: (1) resolve-model.ts: `getGlobalProvider() = asProviderV4(globalThis.AI_SDK_DEFAULT_PROVIDER ?? gateway)`; string model ids in resolveLanguageModel/resolveEmbeddingModel/resolveImageModel/resolveSpeechModel/resolveTranscriptionModel/resolveRerankingModel/resolveVideoModel call provider.<kind>Model(id) on it — so the gateway singleton is the hard default global provider, constructed at module load of 'ai'. (2) util/retry-with-exponential-backoff.ts: shouldRetry includes GatewayError.isInstance(error) && error.isRetryable===true; delay logic reads APICallError.responseHeaders possibly from error.cause. (3) prompt/wrap-gateway-error.ts: GatewayAuthenticationError.isInstance check; rewrites into 'set AI_GATEWAY_API_KEY' guidance (prod: AISDKError name 'GatewayError' if process.env.NODE_ENV==='production'; else ANSI-colored Error) — invoked in generate-text.ts:1454, stream-text.ts:1139, generate-object.ts:499, stream-object.ts:530. (4) types/language-model.ts: GlobalProviderModelId defaults to GatewayModelId (a ~large string-literal union of 'vendor/model' slugs) unless RegisteredProviderModels is declaration-merged. (5) index.ts re-exports createGateway, gateway, type GatewayModelId. Plus generate-image.ts:223 special-cases merging providerMetadata entries keyed 'gateway'. global.ts only declares globalThis.AI_SDK_DEFAULT_PROVIDER/AI_SDK_LOG_WARNINGS/AI_SDK_TELEMETRY_INTEGRATIONS — no gateway import.

## Port implications
The core `ai` port needs only a narrow gateway contract, so a split design works: (1) a tiny `GatewayError` surface in core — a Zig error/diagnostic struct with {status_code: u16, error_type: enum, is_retryable: bool, generation_id: ?[]u8, message} — because retry logic and wrapGatewayError only test isInstance + isRetryable + auth-error kind; in Zig this is naturally a tagged union or a diagnostic struct attached to an error code, no class hierarchy or symbol markers needed. (2) The full gateway provider as a compile-time-optional module (build option or comptime-selected default provider): TS's `globalThis.AI_SDK_DEFAULT_PROVIDER ?? gateway` global fallback should become an explicit registration/vtable pointer (settable default ProviderV4-equivalent), avoiding a hard link dependency. The full port is unusually cheap and high-leverage: the wire body for /language-model IS the JSON serialization of the normalized CallOptions, and the SSE stream IS normalized StreamParts — so the port's canonical (de)serializers for CallOptions/StreamPart double as the gateway codec, and doStream is a pure SSE decode loop over std.Io (skip 'raw' unless includeRawChunks, inject stream-start warnings, parse response-metadata timestamp). Hard/nuanced parts: async per-request header resolution (Resolvable -> a headers callback receiving Io/allocator; OIDC = read VERCEL_OIDC_TOKEN env; skip @vercel/oidc request-context magic or expose a hook for the C FFI layer); stale-while-revalidate + single-flight metadata cache (mutex + generation counter instead of a shared Promise); base64 file encoding must copy rather than mutate caller's prompt (TS mutates in place — do NOT replicate); undici timeout-code sniffing maps to Zig http client timeout error sets; realtime needs a WebSocket client with subprotocol auth (browser constraint irrelevant in Zig but the subprotocol contract must match: 'ai-gateway-realtime.v1', 'ai-gateway-auth.<token>', base64url team). Env vars to support: AI_GATEWAY_API_KEY, VERCEL_OIDC_TOKEN, VERCEL_DEPLOYMENT_ID/ENV/REGION/PROJECT_ID. Headers/protocol constants: 'ai-gateway-protocol-version: 0.0.1', 'ai-gateway-auth-method', 'x-vercel-ai-gateway-team', 'ai-language-model-{specification-version,id,streaming}', 'ai-model-id'. GatewayModelId's giant string-literal union should become plain []const u8 (optionally a comptime-generated enum for known slugs).
