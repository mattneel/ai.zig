# gap:Realtime subsystem end-to-end + WebSocket client for Zig

## Summary
Mapped the full AI SDK realtime stack and the Zig 0.16 substrate for porting it. SDK side: AbstractRealtimeSession is a thin orchestrator over three components — RealtimeEventReducer (pure-ish state machine: RealtimeState {status, messages: UIMessage[], events ring, isCapturing, isPlaying} + 7 side maps, returning {state, effects[]} with effects play-audio/speech-started/tool-call/error), BrowserRealtimeTransport (browser WebSocket + per-model codec via RealtimeModelV4: getWebSocketConfig/parseServerEvent/serializeClientEvent/buildSessionConfig/getHealthCheckResponse), and BrowserRealtimeAudio (AudioContext, browser-only). Tool calls are gated: a single response-create is sent only after response-done closes the tool-bearing turn AND every callId has an output. Auth is uniformly ephemeral-token-over-Sec-WebSocket-Protocol (OpenAI: ['realtime','openai-insecure-api-key.<t>'] to wss://host/v1/realtime?model=..., token minted via POST /realtime/client_secrets; Gateway: ['ai-gateway-realtime.v1','ai-gateway-auth.<t>','ai-gateway-team.<b64url>'] to ws(s)://.../realtime-model?ai-model-id=..., vcst_ secret minted server-side). provider-utils/websocket.ts abstracts the constructor (url, protocols, {headers}) with globalThis fallback. Zig side: std.http.Server.zig has a server-only RFC6455 codec (packed Header0/Header1, readSmallMessage requires masked+fin single frames, writeMessage unmasked, SHA1+GUID accept); std.http.Client has NO WebSocket but maps ws/wss schemes, sends arbitrary extra_headers, returns 101 heads immediately with the (TLS-transparent) connection *Io.Reader parked at the first frame, and has a documented connection-takeover precedent (CONNECT tunnel). ziex confirms the patterns: std-only WS serving with a dedicated frame-drain thread and futex+ping keepalive; its native WS client delegates to websocket.zig rather than std. A complete wss client recipe over http.Client + tls.Client was derived with exact signatures.

## Key findings
### AbstractRealtimeSession public surface (realtime-session.ts)
abstract class AbstractRealtimeSession, options: {model: RealtimeModel, api: {token: string}, sessionConfig?: Partial<RealtimeSessionConfig>, sampleRate?: number (default 24000), maxEvents?: number (default 500), onToolCall?: ({toolCall:{toolCallId,toolName,args}}) => Promise<unknown>|unknown|undefined, onEvent?: (RealtimeServerEvent)=>void, onError?: (Error)=>void}. Public methods: connect(): Promise<void> (POSTs api.token URL with {sessionConfig} JSON, expects RealtimeSetupResponse {token,url,expiresAt?,tools}, then transport.connect and sends {type:'session-update',config} onOpen); disconnect(); sendEvent(RealtimeClientEvent); sendTextMessage(text) (conversation-item-create + response-create + reducer.addUserTextMessage); sendAudio(base64) (input-audio-append); commitAudio(); clearAudioBuffer(); requestResponse({modalities?}); cancelResponse(); addToolOutput(callId, result); startAudioCapture(MediaStream); stopAudioCapture(); stopPlayback(); dispose(). One abstract method: protected abstract setState<K extends keyof RealtimeState>(key,value) — subclasses (e.g. React RealtimeStore in packages/react/src/use-realtime.ts) push per-key changes to UI; applyState() diffs prev/next per key (status, messages, events, isCapturing, isPlaying) and calls setState only on identity change.

### Multi-tool-call response gating (session)
Session tracks private toolCallsInResponse: Set<string>, submittedToolOutputs: Set<string>, responseToolCallsClosed: bool. On reducer effect 'tool-call' it adds callId to toolCallsInResponse and calls executeToolCall (onToolCall handler; returning undefined = human-in-the-loop, output submitted later via addToolOutput; returned value auto-submits). addToolOutput sends conversation-item-create{type:'function-call-output',callId,name,output:JSON.stringify(result)}, adds to submittedToolOutputs, then maybeRequestToolResponse. On server event 'response-done' with toolCallsInResponse.size>0, sets responseToolCallsClosed=true. maybeRequestToolResponse sends a single {type:'response-create'} only when responseToolCallsClosed && every callId in toolCallsInResponse ∈ submittedToolOutputs, then clears all three. Also on 'speech-started' effect while isPlaying: computes playedMs=audio.getPlaybackOffsetMs(), stops playback, sends conversation-item-truncate{itemId:currentResponseItemId, contentIndex:0, audioEndMs:round(playedMs)} (barge-in truncation).

### RealtimeEventReducer state machine
State: {status: 'disconnected'|'connecting'|'connected'|'error', messages: UIMessage[], events: RealtimeServerEvent[] (ring capped at maxEvents=500 via slice(-maxEvents)), isCapturing: bool, isPlaying: bool}. Instance side-maps (mutable, outside immutable state): currentAssistantMessageId: string|null, textAccumulators: Map<itemId,string>, toolArgAccumulators: Map<callId,string>, toolCallIdToMessageId: Map, toolCallIdToName: Map, inputAudioMessageInsertIndex: Map<itemId,number>, itemIdToPartLocation: Map<itemId,{messageId,partIndex}>. reduceServerEvent(state,event) -> {state, effects: RealtimeReducerEffect[]} where effect = {type:'play-audio',itemId,delta} | {type:'speech-started'} | {type:'tool-call',callId,name,args:Record,rawArguments:string} | {type:'error',error}. Transitions: session-created/session-updated: connecting->connected; audio-delta -> play-audio effect; audio-committed: record messages.length as insert index for itemId; audio-transcript-delta/text-delta -> appendTextDelta (get-or-create assistant message id `assistant-${Date.now()}-${rand4}`, accumulate per itemId, part state 'streaming', record part location); audio-transcript-done(transcript)/text-done(text) -> finalizeText (state 'done', delete accumulators/location); input-transcription-completed -> user message id `user-${itemId}` inserted at recorded index (or replace existing); response-created/response-done/speech-started -> currentAssistantMessageId=null (speech-started also emits effect); function-call-arguments-delta -> ensure assistant message + ensureToolPart (DynamicToolUIPart {type:'dynamic-tool', toolName:'', toolCallId, state:'input-streaming', input:undefined}), accumulate args; function-call-arguments-done -> safeParseJSON(arguments), markToolInputAvailable (state:'input-available', toolName, input) then emit tool-call effect (or error effect on parse failure); error -> error effect. addToolOutput -> updateToolPartState (state:'output-available', output) + returns {callId, name: toolCallIdToName.get(callId), output: JSON.stringify(result)}.

### Transport abstraction contract (BrowserRealtimeTransport + RealtimeModelV4)
BrowserRealtimeTransport options: {model: RealtimeModel, onServerEvent: (e)=>void|Promise<void>, onError: (Error)=>void, onClose: ()=>void}. connect({token,url,onOpen}): closes old ws, calls model.getWebSocketConfig({token,url}) -> {url, protocols?: string[]}, new WebSocket(url, protocols) (browser global; no headers possible). Tracks ws before onopen so disconnect() during CONNECTING works; stale-socket guards (this.ws !== ws) on open/close. sendEvent(event) serializes through a promise chain sendQueue to preserve order across async model.serializeClientEvent(event); null serialization = skip. sendRaw sends string/ArrayBuffer/view/Blob as-is else JSON.stringify. handleMessage: decode text (string|Blob.text()|TextDecoder), safeParseJSON, then optional model.getHealthCheckResponse(rawEvent) -> auto-reply raw (provider app-level keepalive), then model.parseServerEvent(raw) returning one event or array, each awaited through onServerEvent. RealtimeModelV4 interface: {specificationVersion:'v4', provider: string, modelId: string, doCreateClientSecret(opts:{expiresAfterSeconds?, sessionConfig?}) -> PromiseLike<{token,url,expiresAt?}> (server-side), getWebSocketConfig({token,url})->{url,protocols?}, parseServerEvent(raw)->Event|Event[], serializeClientEvent(event)->unknown|PromiseLike, buildSessionConfig(config)->unknown, getHealthCheckResponse?(raw)->unknown|null}. RealtimeFactoryV4: callable (modelId)->RealtimeModelV4 plus getToken({model, expiresAfterSeconds?, sessionConfig?}) -> Promise<{token,url,expiresAt?}>.

WebSocketLike = {readyState: number, send(data: string|Uint8Array|ArrayBuffer): void, close(code?: number, reason?: string): void, onopen/onmessage/onerror/onclose: nullable callbacks (onmessage gets {data: unknown})}. ### Browser-only vs portable split
Portable (pure logic, direct Zig port): realtime-event-reducer.ts, realtime-session.ts (minus audio/transport instantiation), realtime-types.ts, get-realtime-tool-definitions.ts (iterates ToolSet; skips type:'provider' tools; resolves description string-or-function; parameters = await asSchema(tool.inputSchema).jsonSchema), audio-utils.ts (encodeRealtimeAudio: f32 [-1,1] -> PCM16 LE -> base64; decodeRealtimeAudio: reverse, /32768.0; resampleAudio: linear interpolation, ratio=inRate/outRate, outputLength=round(len/ratio)). Browser-only: browser-realtime-audio.ts (AudioContext, ScriptProcessorNode(4096,1,1), MediaStreamAudioSourceNode, AudioBufferSourceNode playback queue with playbackTime scheduling, getPlaybackOffsetMs=(currentTime-playbackStartTime)*1000) and browser-realtime-transport.ts's use of global WebSocket + Blob. ### std.http.Server.zig WebSocket (server-side only, Zig 0.16)
Request.upgradeRequested(request: *const Request) UpgradeRequest where UpgradeRequest = union(enum){websocket: ?[]const u8 (sec-websocket-key), other: []const u8, none} — HTTP/1.1 GET only. Request.respondWebSocket(options: WebSocketOptions{key, reason?: ?[]const u8, extra_headers: []const http.Header}) ExpectContinueError!WebSocket — writes '101 ... connection: upgrade\r\nupgrade: websocket\r\nsec-websocket-accept: ' + base64.standard(SHA1(key ++ "258EAFA5-E914-47DA-95CA-C5AB0DC85B11")) (28 bytes); not flushed until WebSocket.flush. pub const WebSocket = struct {key: []const u8, input: *Io.Reader, output: *Io.Writer} with Header0 = packed struct(u8){opcode: Opcode(u4), rsv3/rsv2/rsv1: u1=0, fin: bool}, Header1 = packed struct(u8){payload_len: enum(u7){len16=126,len64=127,_}, mask: bool}, Opcode = enum(u4){continuation=0,text=1,binary=2,connection_close=8,ping=9,pong=10,_}. readSmallMessage() ReadSmallTextMessageError!SmallMessage{opcode,data:[]u8}: takeArray(2), REQUIRES h1.mask (error.MissingMaskBit — server rule), REQUIRES fin (error.MessageOversize — no fragmentation), len via takeInt(u16/u64,.big), payload must fit in.buffer (error.MessageOversize), unmasks in place via u32 XOR over word-aligned prefix + byte tail, close frame -> error.ConnectionClose (code/reason not parsed), pong skipped, ping returned to caller (no auto-pong). writeMessage/writeMessageUnflushed/writeMessageVec/writeMessageVecUnflushed(data, op) Writer.Error!void: always fin=true, mask=false (server-side), length encoding 0..125 inline / 126 u16 / 127 u64 big-endian, writeVecAll payload. So: reusable pieces for a client = header packed structs, opcode enum, length encode/decode; NOT reusable as-is = mask direction (client must SEND masked, RECEIVE unmasked — exact inverse), no close handshake, no fragmentation, no client handshake.

### std.http.Client: no WebSocket, but complete Upgrade plumbing exists
Zero websocket code in Client.zig, BUT Protocol.fromScheme maps {'http':.plain,'ws':.plain,'https':.tls,'wss':.tls} so client.request() accepts wss:// URIs directly. Signatures: pub fn request(client: *Client, method: http.Method, uri: Uri, options: RequestOptions) RequestError!Request; RequestOptions {version=.@"HTTP/1.1", handle_continue=true, keep_alive=true, redirect_behavior: Request.RedirectBehavior=@enumFromInt(3), connection: ?*Connection=null, headers: Request.Headers (host/authorization/user_agent/connection/accept_encoding/content_type each Value=union(enum){default,omit,override:[]const u8}), extra_headers: []const http.Header, privileged_headers}. Request.sendBodiless() Writer.Error!void; Request.receiveHead(redirect_buffer: []u8) ReceiveHeadError!Response. Critical control flow: 1xx informational responses (101!) return from receiveHead immediately with no body framing ('r.method == .HEAD or head.status.class() == .informational ...' branch), leaving connection.reader() positioned exactly at the first WebSocket frame (http.Reader.receiveHead tosses only the head bytes; surplus stays buffered in the same *Io.Reader). Response.Head {bytes, version, status: http.Status, reason, location?, content_type?, keep_alive: bool (Connection: close => false; 'upgrade' => true), content_length?, transfer_encoding, content_encoding} + iterateHeaders() http.HeaderIterator for reading sec-websocket-accept/protocol. Connection takeover precedent = connectProxied CONNECT tunnel: after success set 'req.connection = null' so Request.deinit doesn't release it to the pool; Connection.reader()/writer() return *Io.Reader/*Io.Writer that route through std.crypto.tls.Client (fields tls.client.reader/.writer) for .tls or raw stream interfaces for .plain; Connection.flush(), Connection.end() (sends TLS close_notify), Connection.destroy(io) closes socket and frees. Client struct: {allocator, io: std.Io, ca_bundle, tls_buffer_size=tls.Client.min_buffer_len, connection_pool, read_buffer_size=8192, write_buffer_size=1024, http_proxy/https_proxy}. connectTcp(client,host: HostName,port,protocol) ConnectTcpError!*Connection. NOTE: default 8192 read buffer bounds any borrowed-slice message API; realtime audio-delta JSON frames (base64 PCM) can exceed it, so the WS client needs a larger buffer or a streaming message reader.

### std.crypto.tls.Client shape (for direct wss use)
Fields: input: *Io.Reader (encrypted from server; buffer >= min_buffer_len asserted), reader: Io.Reader (decrypted), output: *Io.Writer (encrypted to server), writer: Io.Writer (plaintext), alert: ?tls.Alert, read_err: ?ReadError, received_close_notify: bool. pub const min_buffer_len = tls.max_ciphertext_record_len. pub fn init(input: *Reader, output: *Writer, options: Options) InitError!Client where Options = {host: union(enum){no_verification, explicit: []const u8}, ca: union(enum){no_verification, self_signed, bundle: {gpa, io, lock: *Io.RwLock, bundle: *Certificate.Bundle}}, write_buffer: []u8, read_buffer: []u8, entropy: *const [240]u8 (fill via io.random), realtime_now: Io.Timestamp, ssl_key_log: ?*SslKeyLog=null, allow_truncation_attacks: bool=false, alert: ?*tls.Alert=null}. pub fn end(c: *Client) Writer.Error!void sends close_notify. http.Client wraps all of this in Connection.Tls (single aligned allocation: struct + host + tls_read(buffer_size+tls_buffer) + tls_write + socket bufs), so the easiest wss path is going through http.Client rather than hand-rolling TLS.

### std.Io.net.Stream reader/writer (Io model)
Stream = struct{socket: Socket} with close(io), shutdown(io, how). Stream.Reader = {io: Io, interface: Io.Reader, stream, err: ?Error} constructed via stream.reader(io, buffer: []u8); vtable {stream, readVec} calls io.vtable.netRead; 0 bytes => error.EndOfStream; detailed err parked in .err with error.ReadFailed surfaced (same pattern as tls read_err and http Connection.getReadError()). Stream.Writer = {io, interface: Io.Writer, stream, err, write_file_err} via stream.writer(io, buffer); drain -> io.vtable.netWrite; sendFile unimplemented. All higher layers (http, tls, a future ws) are pure *Io.Reader/*Io.Writer adapters — the WS frame codec should follow the identical pattern: struct with interface: Io.Reader/Io.Writer or plain methods over *Io.Reader/*Io.Writer.

### ziex real-world patterns (DevServer.zig, runtime websocket)
DevServer (std-only, server side): per-connection std.Thread; stream.reader(io,&recv_buf)/writer(io,&send_buf) -> http.Server.init(&reader.interface,&writer.interface); loop server.receiveHead(); switch request.upgradeRequested() {.websocket => |key| respondWebSocket(.{.key=key}) then serveWebSocket, .other, .none}. serveWebSocket: first writeMessage also flushes the 101; spawns DEDICATED recv thread recvWebSocketFrames that loops sock.readSmallMessage() purely to drain pings/close and keep the TCP window open ('Without this, the connection stalls'); main loop blocks on futexWaitTimeout over an atomic update_id with 30s timeout -> on timeout sends sock.writeMessage("", .ping) as keepalive; bounded event ring (16) under Io.Mutex with lockUncancelable(io); events serialized via std.json.fmt. ziex native WS *client* does NOT hand-roll frames: runtime/server/websocket.zig wraps third-party websocket.zig (via httpz): websocket.Client.init(allocator,{host,port,tls}), client.handshake(path,.{}), client.readLoop(&handler) on std.Thread with handler methods serverMessage(data, msg_type)/serverClose(data: close code = readInt(u16,data[0..2],.big), reason=data[2..])/close(); writeText/writeBin take MUTABLE []u8 (library masks in place — confirming client-side masking mutates the payload buffer). runtime/client/websocket.zig is the WASM/browser FFI shim: extern _wsConnect(ws_id,url_ptr,len,protocols_ptr,len)/_wsSend(id,ptr,len,is_binary)/_wsClose(id,code,reason), exported callbacks __zx_ws_onopen/onmessage/onerror/onclose(ws_id,...) dispatched through a fixed 32-slot registry keyed by u64 id — a direct template for the ai.zig C-ABI callback surface. core/WebSocket.zig defines the MDN-style portable facade: ReadyState enum(u16){connecting=0,open=1,closing=2,closed=3}, CloseOptions{code:?u16 (1000|3000-4999), reason max 123 bytes}, MessageEvent{data: union(enum){text,binary}}, handler fn-pointer fields, comptime platform dispatch (is_wasm ? client_impl : server_impl).

### Concrete wss:// client recipe over std (synthesis)
(1) Handshake via std.http.Client: var req = try client.request(.GET, uri /* wss:// works: Protocol.fromUri */, .{ .redirect_behavior = .unhandled, .headers = .{ .connection = .{ .override = "Upgrade" }, .accept_encoding = .omit }, .extra_headers = &[_]http.Header{ .{.name="upgrade",.value="websocket"}, .{.name="sec-websocket-version",.value="13"}, .{.name="sec-websocket-key",.value=&key24}, .{.name="sec-websocket-protocol",.value=joined_protocols} } }); try req.sendBodiless(); const resp = try req.receiveHead(&.{}); Sec-WebSocket-Key: var raw:[16]u8; io.random(&raw) (or std.crypto.random); var key24:[24]u8; _=std.base64.standard.Encoder.encode(&key24,&raw). (2) Validate: resp.head.status == .switching_protocols (101 returns immediately from receiveHead, connection reader positioned at first frame); iterate resp.head.iterateHeaders() for 'sec-websocket-accept' == base64(SHA1(key24 ++ "258EAFA5-E914-47DA-95CA-C5AB0DC85B11")) (28 chars, compute like Server.respondWebSocket), capture 'sec-websocket-protocol' (must be one offered, e.g. 'ai-gateway-realtime.v1' or 'realtime'). (3) Takeover: const conn = req.connection.?; req.connection = null; req.deinit(); use conn.reader()/conn.writer() (*Io.Reader/*Io.Writer, TLS-transparent); keep *Connection to conn.end()+destroy() later. (4) Frame writer (client): reuse Header0/Header1 packed structs from http.Server.WebSocket but set Header1.mask=true, write 4-byte random masking key, then XOR payload while copying into the output writer (cannot writeVecAll borrowed const data; mask per-chunk into writer.writableSlice or a scratch buffer); new key per frame; fin=true for unfragmented. (5) Frame reader (client): inverse of readSmallMessage — reject h1.mask==true from server (protocol error), accept fin/continuation for 1:1 fidelity or error like std; opcodes: text/binary deliver; ping (payload<=125) => immediately write masked pong echoing payload; pong => reset keepalive deadline; connection_close => parse optional code (u16 big-endian) + reason, echo masked close once, then conn.end(). For large realtime messages (base64 audio deltas), stream payload into an allocator-backed ArrayList with a max_message_size limit instead of borrowing the fixed reader buffer. (6) Close initiation: send masked close{code,reason<=123}, await peer close or timeout, then TLS close_notify via conn.end(), conn.destroy(io). (7) Concurrency: dedicated recv loop (Io.async / std.Thread per DevServer pattern), writer protected by Io.Mutex because pong/close/keepalive-ping interleave with sendEvent frames; 30s ping keepalive via timeout on the event wait (futexWaitTimeout pattern) — distinct from provider-level getHealthCheckResponse JSON keepalive which stays in the transport layer.

## Port implications
The realtime stack ports cleanly into four layers. (a) Portable core: RealtimeEventReducer becomes a struct with mutable RealtimeState + side HashMaps (StringHashMap for itemId/callId maps); reduceServerEvent returns effects as a bounded ArrayList of a tagged union (play_audio/speech_started/tool_call/error) — TS's immutable-copy state diffing collapses to per-field dirty flags feeding an abstract setState callback (the C-ABI hook, mirroring ziex's __zx_ws_* exported-callback + slot-registry pattern for Python/Rust wrappers). Client/server events are natural Zig tagged unions; every server event carries raw: keep the parsed std.json.Value alive per event (arena per message). RealtimeModelV4 maps to a vtable interface with 6 fns (doCreateClientSecret, getWebSocketConfig, parseServerEvent, serializeClientEvent, buildSessionConfig, optional getHealthCheckResponse); Gateway model is an identity codec, OpenAI model is a pure JSON mapper — both allocation-light. (b) Transport: define a RealtimeTransport interface (connect/sendEvent/sendRaw/disconnect + onServerEvent/onError/onClose callbacks) — TS hard-wires BrowserRealtimeTransport into the session constructor; invert that. The promise-chain sendQueue becomes a mutex-guarded writer or a queue task. (c) The hard part is the wss client, which must be written from scratch (~500 lines): std.http.Client has zero WebSocket support but already accepts ws/wss schemes, does TLS via std.crypto.tls.Client internally, and its receiveHead returns 101 responses with the connection *Io.Reader positioned at the first frame — so the handshake is request(.GET)+extra_headers+sendBodiless+receiveHead, then steal the Connection (req.connection = null, the CONNECT-tunnel precedent). Reuse std.http.Server.WebSocket's Header0/Header1 packed structs and Sec-WebSocket-Accept SHA1 recipe, but invert masking (client sends masked — payload must be XOR-copied, not writeVec'd; server frames must be unmasked), add ping→pong auto-reply, close handshake with code/reason, and an allocator-backed message reader because base64 audio deltas exceed the 8 KiB default buffer that readSmallMessage-style borrowing allows. Auth never uses headers: OpenAI/Gateway tokens ride Sec-WebSocket-Protocol subprotocols, so the client must send Sec-WebSocket-Protocol and surface the negotiated protocol. Run a dedicated recv loop (Io.async or thread, per ziex DevServer) plus 30s ping keepalive via wait-with-timeout.
