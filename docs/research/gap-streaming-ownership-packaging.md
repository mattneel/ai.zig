# gap:Zig streaming-architecture mapping: WHATWG semantics onto Io primitives, allocator/ownership strategy, and build.zig packaging

## Summary
Researched Zig 0.16 std.Io streaming primitives, stdlib memory-ownership patterns, and multi-module packaging, all from source. Io.Queue(Elem) (Io.zig:2184) is a many-producer/many-consumer bounded ring over a caller-provided buffer with blocking backpressure on both sides, close() semantics that drain-then-error, and full cancelation integration — but delivery is exactly-once per element (work-queue), so tee/fan-out does NOT exist anywhere in std.Io; a broadcast primitive must be built (two viable designs sketched, one matching ziex's real pubsub.zig RwLock+iterate-subscribers pattern, one matching Web-streams tee via shared log + per-consumer cursors). DelayedPromise maps to Io.Event (futex-based one-shot flag, Io.zig:1766) plus a value slot, not to Future (Future requires a spawned task and io.async may execute eagerly/inline). Stitchable stream maps to an outer Io.Queue(Part) + Io.Queue of inner-stream handles + a driver task in an Io.Group; Queue.close gives exactly the SDK's close-vs-terminate split. std.http.Client uses a thread-safe gpa for pooled connections (single slab alloc per connection embedding all buffers), caller-provided transfer/decompress buffers for bodies, and borrow-until-invalidated header strings; std.json uses Parsed(T) = arena-per-document and Token slices that either borrow the scanner buffer or are explicitly allocated (alloc_if_needed/alloc_always). This grounds an arena-per-step ownership contract for stream parts with copy-out (or clone) at the C FFI boundary using borrow-until-next-call pull semantics. Packaging: one repo exposes N named modules via b.addModule; consumers use dep.module("name"); static+shared libs from two addLibrary calls sharing a root module, headers via installHeader/installHeadersDirectory; per-module tests are separate b.addTest(.{.root_module=...}) steps with filters.

## Key findings
### Io.Queue init/capacity
pub fn Queue(Elem: type) wraps TypeErasedQueue (Io.zig:2184, 1874). init(buffer: []Elem) — capacity is exactly the caller-provided buffer length (runtime-configurable, bounded, zero internal allocation; capacity 0 is legal → pure rendezvous through putter/getter wait lists). Internally a byte ring buffer + std.DoublyLinkedList of blocked Put{remaining, needed, condition, node} / Get records; typed API does @ptrCast + @divExact(sizeOf) over the byte queue, so elements are memcpy'd by value (slices inside elements are shallow-copied — payload lifetime is the port's responsibility).

### Io.Queue blocking/backpressure semantics
put(q, io, elements, min) (QueueClosedError||Cancelable)!usize: copies directly into waiting getters first, then ring buffer, blocks on a per-request Condition only if fewer than min transferred; min=0 = nonblocking best-effort. get symmetric. putAll = put with min=len. putOne/getOne single-element helpers. All have *Uncancelable variants that are not cancelation points. Backpressure is inherent: full buffer + min>satisfied ⇒ producer suspends until a consumer drains (getLocked calls fillRingBufferFromPutters to resume putters, Io.zig:2159).

### Io.Queue close/cancel semantics
close(io) (Io.zig:1918): idempotent, threadsafe; signals all blocked putters+getters; puts fail error.Closed immediately even with space; gets drain remaining buffered elements first, then return error.Closed (drain-then-fail — exactly SSE 'flush then done'). Cancel: if canceled mid-wait with partial progress, put/get return the partial count and io.recancel() re-arms so the NEXT Io call returns error.Canceled; guaranteed error.Canceled/Closed on subsequent calls.

### single vs multi consumer; no tee exists
Doc comment Io.zig:2181: 'Many producer, many consumer, thread-safe'. But each element is delivered to exactly one getter (getters queue is FIFO; putLocked pops getters and fills them) — work-distribution semantics, NOT broadcast. Grepped all of Io.zig + Io/ subdir: no tee/broadcast/fan-out primitive exists (only Condition.broadcast, Io.zig:1745, which wakes all waiters of a condvar). A broadcast queue must be built in ai.zig.

### broadcast queue sketch (two grounded designs)
(a) subscriber-list fan-out, matching real-world ziex src/runtime/server/pubsub.zig: registry {std.Io.RwLock, list of *Subscriber}, publish takes lockShared and putOne's into each subscriber's own Io.Queue(T) (or writes to conn directly); slowest subscriber blocks publisher (or use min=0 put + drop policy). (b) shared append-only log + per-consumer cursor: ArrayList(Part) guarded by Io.Mutex + Condition; append → cond.broadcast(io); each consumer reads log[cursor..] at its own pace; memory = lag, which streamText already pays because it accumulates parts for the final result — this matches Web ReadableStream.tee() unbounded-buffer semantics and is the better fit for the SDK's teeStream usage.

### Future/async/concurrent (NOT lazy)
Future(Result) = struct{ any_future: ?*AnyFuture, result: Result } with await(f,io)/cancel(f,io) → Result, idempotent, not threadsafe (Io.zig:1176). io.async(function, args) Future(R) MAY run inline before returning (Threaded runs inline when pool exhausted); io.concurrent → ConcurrentError!Future guarantees real concurrency or error.ConcurrencyUnavailable. cancel() = await + request cancelation; next cancelation point in task returns error.Canceled (one-shot, re-armed via io.recancel; maskable via swapCancelProtection(.blocked), Io.zig:1322).

### DelayedPromise equivalent
SDK DelayedPromise (provider-utils/src/delayed-promise.ts) is a one-shot externally-resolved cell, not a task — the Zig equivalent is NOT Future but Io.Event (Io.zig:1766, enum(u32){unset,waiting,is_set}, futex-based): struct { event: Io.Event = .unset, value: E!T = undefined; fn resolve(self,io,v){self.value=v; self.event.set(io);} fn await(self,io) Cancelable!(E!T){try self.event.wait(io); return self.value;} }. Event supports multiple concurrent waiters, set() releases memory ordering for the value write, resolve-before-await works (wait returns immediately if is_set). The JS 'lazy construction to avoid unhandled rejection' motivation vanishes in Zig.

### stitchable stream equivalent
createStitchableStream (ai/src/util/create-stitchable-stream.ts) maps to: outer Io.Queue(Part) read by consumer; inner Io.Queue(*InnerStream) of pending streams; driver task spawned via Io.Group.async/concurrent loops { inner = inner_q.getOne() catch break; while (part = inner.next()) outer.putOne(part); } then outer.close(io). addStream = putOne to inner_q; close() = inner_q.close(io) (drain remaining steps then outer closes — graceful); terminate() = group.cancel(io) + outer.close(io) (immediate). Queue.close's put-fails-immediately rule reproduces the SDK's 'Cannot add inner stream: outer stream is closed' throw as error.Closed. Io.Select(U) (Io.zig:1367, = Group + Queue(U) of tagged-union results) is the tool for racing abort/timeout against stream progress.

### stdlib producer-into-queue idiom
The Io VTable itself streams results via caller-owned queues: netLookup: *const fn (?*anyopaque, net.HostName, *Queue(net.HostName.LookupResult), net.HostName.LookupOptions) net.HostName.LookupError!void (Io.zig:254) — producer fills a *Io.Queue out-param owned by the caller. This is the sanctioned shape for ai.zig's doStream: provider writes LanguageModelStreamPart into a caller-provided *Io.Queue(StreamPart).

### std.http.Client memory model
Client{ allocator: Allocator /* must be thread-safe */, io: Io, connection_pool, read_buffer_size=8192, write_buffer_size=1024 } (Client.zig:27-55). Each Connection is ONE slab allocation embedding struct+hostname+all reader/writer buffers (Plain.create Client.zig:249-291; Tls.create adds tls_buffer_size-sized TLS frames buffers, :299-388). Request bodies: Response.reader(transfer_buffer: []u8) *Io.Reader (Client.zig:736) — caller supplies the transfer buffer; readerDecompressing takes caller decompress buffers; fetch() heap-allocates redirect/decompress buffers from client.allocator only when options leave them null (Client.zig:1831-1851) and streams body into a caller *Writer via reader.streamRemaining. Header strings (head.location/content_type) borrow the connection read buffer and are explicitly invalidated (invalidateStrings, Client.zig:718) once body reading starts — borrow-until-state-change is the stdlib norm; no arena-per-request anywhere in http.Client.

### std.json memory model
Parsed(T) = { arena: *ArenaAllocator, value: T } with deinit destroying the arena (static.zig:56-67); parseFromTokenSource creates arena from gpa, parses into it (static.zig:102-119); *Leaky variants parse into any allocator (caller brings arena). Streaming: json.Reader = { scanner: Scanner.initStreaming(alloc), reader: *std.Io.Reader } (Scanner.zig:1574-1584); Token is a union: .string/.number are SLICES INTO THE SCANNER BUFFER (valid only until next() call), .allocated_string/.allocated_number: []u8 are caller-allocator-owned copies; AllocWhen{alloc_if_needed, alloc_always} chooses; with a streaming *json.Reader the default is .alloc_always because buffer contents shift (static.zig:141-147). max_value_len default 4MiB guard.

### proposed ownership contract for the 5-stage stream pipeline
Grounded synthesis: stage 1 (socket bytes) live in Client-owned connection buffer; stage 2 (SSE event framing) borrows that buffer; stage 3 (provider JSON parse) MUST copy out — parse each SSE data payload with parseFromTokenSourceLeaky into an arena; stage 4/5 (provider part → SDK part → consumer queue) pass structs by value through Io.Queue with string fields pointing into that arena. Recommended: one ArenaAllocator per streamText STEP (matches Parsed(T) precedent and stitchable-stream step boundaries); all parts of a step share it; freed after step-finish part is consumed AND the step's accumulated text is copied into the whole-call result arena. Refcounting is unnecessary for the single-consumer path; for tee'd consumers use design (b) shared-log broadcast where the log itself keeps the arena alive until every cursor passes end-of-step. Rule: a part's slices are valid until the arena that produced them is reset; crossing any boundary that outlives the step ⇒ dupe into the destination's allocator.

### C FFI chunk ownership (reconciling callee-allocated out-params)
Callee-allocated out-params per chunk (one malloc + free per delta) is the wrong shape at stream frequency. stdlib precedent (json Token borrow-until-next-call; http head strings borrow-until-invalidate) supports a pull API: `ai_stream_next(stream: *AiStream, out: *ai_stream_part_t) -> ai_status` where out's pointers borrow an internal per-stream scratch arena reset at the START of the next ai_stream_next call (single-owner, zero-copy for well-behaved consumers); provide `ai_stream_part_clone(part, *owned)` + `ai_part_free` for retention, and keep callee-allocated out-params only for one-shot results (generateText final). This maps directly onto the internal Io.Queue: FFI next() = queue.getOne(io) + arena reset + translate; error.Closed → AI_STREAM_DONE; error.Canceled → AI_CANCELED. Blocking next() is fine because the C thread just parks in Threaded futexWait; also expose a poll variant via get(min=0).

### multi-module packaging (mirrors provider/provider-utils/ai/openai/anthropic split)
Two proven layouts, combinable: (1) one build.zig exposing many named modules: b.addModule(name, Module.CreateOptions{.root_source_file, .target, .optimize, .imports: []const .{name,module}, .link_libc, ...}) *Module (Build.zig:909, Module.zig:216-263); intra-repo wiring via mod.addImport(name, other_mod) — ziex/build.zig:57-64 exposes zx_style, zx_core_lang, zx and wires zx_core_lang into zx. (2) path-based sub-packages in one repo: build.zig.zon .dependencies = .{ .adapters = .{ .path = "pkg/adapters" } } where pkg/adapters has its own build.zig.zon (own .name/.fingerprint) and its build.zig addModule's several modules (ziex pkg/adapters exposes db and db_sqlite). Consumers: const dep = b.dependency("ai", .{ .target = target, .optimize = optimize }); mod.addImport("ai", dep.module("ai")); dep.module panics with available-names listing if missing (Build.zig:1961). Transitive access: dep.builder.dependency(...) reaches the dependency's own deps (ziex/site/build.zig:13-14). build.zig.zon itself is importable: @import("build.zig.zon") for version metadata (ziex/build.zig:1). Note b.addModule with same gpa-arena map key: modules registry is b.modules StringHashMap.

### static+shared libs with installed headers
b.addLibrary(LibraryOptions{ .linkage: std.builtin.LinkMode = .static, .name, .root_module: *Module, .version: ?SemanticVersion, ... }) *Step.Compile (Build.zig:823-856). Two calls with .linkage=.static and .linkage=.dynamic can SHARE the same *Module root (Module.CreateOptions has no linkage; linkage is per-Compile), then b.installArtifact(each). Headers: lib.installHeader(source: LazyPath, dest_rel_path: []const u8) and lib.installHeadersDirectory(source, dest_rel_path, HeaderInstallation.Directory.Options) (Step/Compile.zig:523, 537) install to zig-out/include (h_dir, Build.zig:739/758) alongside the artifact AND propagate to any module that links the artifact (addHeaderInstallationToIncludeTree); installLibraryHeaders(lib) forwards another lib's headers; b.addInstallHeaderFile(source, dest_rel_path) (Build.zig:1713) installs without an artifact. For the C ABI lib: root module needs .pic=true for the shared variant if consumed via dlopen on some targets; export fns via export keyword — no extra build API needed.

### per-module `zig build test` filtering
b.addTest(TestOptions{ .name="test", .root_module: *Module, .filters: []const []const u8 = &.{}, .test_runner: ?Step.Compile.TestRunner, .emit_object }) *Step.Compile (Build.zig:858-893); filters become --test-filter args to the test binary (Step/Compile.zig:1374). Pattern from ziex/build.zig:134-177: create one addTest per module (mod_tests, exe_tests, testing_mod_tests — the latter with a custom .test_runner = .{ .path = b.path("test/runner.zig"), .mode = .simple }), wrap each in b.addRunArtifact, and hang them all off b.step("test", ...); ALSO create per-module steps (b.step("test-provider", ...) etc.) so `zig build test-openai` runs one module's tests. Wire user filtering with `.filters = b.args orelse &.{}` or a `-Dtest-filter` b.option passed into .filters.

### Io instantiation for the port + real-world entry pattern
std.Io.Threaded.init(gpa: Allocator /* threadsafe; only used by async/concurrent/groupAsync/groupConcurrent — Allocator.failing legal otherwise */, InitOptions{ .async_limit: ?Io.Limit (default cpuCount-1; when exhausted io.async runs INLINE), .concurrent_limit: Io.Limit = .unlimited (exhausted ⇒ error.ConcurrencyUnavailable), .stack_size, .environ }) Threaded, then .io() → Io (Threaded.zig:1573-1806). ziex App.zig:23 does exactly `var threaded = std.Io.Threaded.init(allocator, .{});`. Io.Evented (fiber-based) exists behind fiber.supported (Io.zig:31). Io.failing (Io.zig:2512) is a complete no-op vtable — ideal for unit-testing ai.zig error paths without network.

## Port implications
Design constraints: (1) Every stream in ai.zig should be an Io.Queue(StreamPart) with a caller-provided or arena-provided buffer — this gives bounded backpressure for free and its close() drain-then-error semantics map 1:1 to ReadableStream close; abort/AbortSignal maps to Group.cancel/Future.cancel with error.Canceled surfacing at the next Io call. (2) Fan-out (fullStream + textStream + accumulation for final result) has NO stdlib primitive; build a small Broadcast(T) as a shared append-only part-log (Io.Mutex + Condition.broadcast + per-consumer cursors) — it doubles as the step accumulator the SDK needs anyway, so 'tee buffering' costs nothing extra. (3) DelayedPromise ports as {Io.Event, E!T value} one-shot cell (usage: streamText's resolved-promises like .text, .usage, .finishReason); stitchable multi-step streaming ports as outer-queue + queue-of-inner-streams + one driver task in an Io.Group. (4) Ownership: arena-per-step for part payloads (json Parsed(T) precedent); parts flow by value through queues with slices into the step arena; dupe only when crossing a lifetime boundary. Text deltas MUST be copied out of the http connection buffer at SSE-parse time (stdlib invalidates borrowed transport slices). (5) C FFI: replace per-chunk callee-allocated out-params with a pull API whose out-struct borrows a per-stream scratch arena valid until the next call (json Token idiom), plus clone/free for retention; blocking next() is safe because Threaded parks the caller's thread in futexWait. (6) Packaging: single repo, one root build.zig exposing modules `provider`, `provider_utils`, `ai`, `openai`, `anthropic`, `gateway` via b.addModule with addImport wiring mirroring the npm dep graph; C ABI artifact via two b.addLibrary(.static/.dynamic) calls over one ffi root module + installHeadersDirectory(\"include\", \"ai\"); per-module test steps (test-provider, test-ai, ...) aggregated under `test`. Hard parts: Queue elements are memcpy'd so tagged-union StreamPart must stay POD-with-slices (no self-references); cancelation is one-shot (must recancel when swallowing partial progress); io.async may run inline (never rely on concurrency without io.concurrent and handle ConcurrencyUnavailable, which is also the single-threaded C-consumer story).
