# sdk-provider-utils

## Summary
provider-utils (v5.0.7) is the HTTP/JSON/streaming plumbing shared by all providers. HTTP layer: postJsonToApi/postFormDataToApi wrap postToApi; getFromApi is the GET twin. Both take url, headers (Record<string,string|undefined>), body {content, values}, failedResponseHandler + successfulResponseHandler (both `ResponseHandler<T>`), optional AbortSignal and injectable FetchFunction (defaults to globalThis.fetch). They append a user-agent suffix (`ai-sdk/provider-utils/<VERSION>` + `runtime/<env>`), extract response headers into a plain lowercased record, and on !response.ok run the failed handler and throw its APICallError; handler exceptions are wrapped in APICallError ("Failed to process error response"/"Failed to process successful response") unless abort/APICallError. Outer catch runs handleFetchError which converts network failures (fetch-failed TypeError with cause, Bun error codes) into retryable APICallError, passing aborts through. SSE: createEventSourceResponseHandler pipes response.body through TextDecoderStream → EventSourceParserStream (eventsource-parser ^3.0.8) → TransformStream that drops `[DONE]` and safeParseJSON-validates each event's `data`, yielding ParseResult<T>. The SSE parser is a spec-compliant line scanner (LF/CR/CRLF, BOM strip, data/event/id/retry fields, comment lines, multi-line data joined with \n, blank line dispatch, partial-line buffering across chunks). createJsonStreamResponseHandler no longer exists in this version. retryWithExponentialBackoff lives in provider-utils (generic engine: maxRetries=2, initialDelay=2000ms, factor=2, pluggable shouldRetry/getDelayInMs/createRetryError); the ai package layers retryWithExponentialBackoffRespectingRetryHeaders on it (retry-after-ms/retry-after parsing, ≤60s cap). JSON: secureJsonParse (proto/constructor poisoning guard), parseJSON/safeParseJSON→schema validation; fixJson (in packages/ai) is the linear-scan partial-JSON repair state machine. Schema abstraction: Schema<T> {jsonSchema, validate?} with asSchema converting Zod/StandardSchema/lazy forms. Config: loadApiKey/loadSetting/loadOptionalSetting read param-or-env. generateId: 16-char random from 62-char alphabet via createIdGenerator.

## Key findings
### postToApi / postJsonToApi / postFormDataToApi
post-to-api.ts. postToApi<T>({url, headers={}, body:{content: string|FormData|Uint8Array, values: unknown}, failedResponseHandler: ResponseHandler<Error>, successfulResponseHandler: ResponseHandler<T>, abortSignal?, fetch = globalThis.fetch}). postJsonToApi sets 'Content-Type: application/json' (caller headers spread AFTER so they can override) and body {content: JSON.stringify(body), values: body} — raw serialized string and structured values both kept (values used only for error reporting). Fetch call: method POST, headers = withUserAgentSuffix(headers, `ai-sdk/provider-utils/${VERSION}`, getRuntimeEnvironmentUserAgent()), body: body.content, signal: abortSignal. On !response.ok: run failedResponseHandler and `throw errorInformation.value`; if the handler itself throws non-abort/non-APICallError, wrap in APICallError{message:'Failed to process error response', statusCode, url, responseHeaders, requestBodyValues}. Same wrapping pattern for successfulResponseHandler ('Failed to process successful response'). Entire body in try/catch that rethrows via handleFetchError. getFromApi (get-from-api.ts) is identical minus body; requestBodyValues={}.

### ResponseHandler type
response-handler.ts line 8: type ResponseHandler<RETURN_TYPE> = (options: {url: string; requestBodyValues: unknown; response: Response}) => PromiseLike<{value: RETURN_TYPE; rawValue?: unknown; responseHeaders?: Record<string,string>}>. Auth is NOT handled here — providers pass Authorization/x-api-key in headers themselves (built with combineHeaders + loadApiKey).

### createJsonResponseHandler / createJsonErrorResponseHandler / binary / status-code handlers
createJsonResponseHandler<T>(responseSchema): reads body as text via readResponseWithSizeLimit (2 GiB default cap; rejects early on Content-Length, cancels body, then incremental read with running byte count), safeParseJSON with schema; on failure throws APICallError('Invalid JSON response'); returns {responseHeaders, value, rawValue}. createJsonErrorResponseHandler({errorSchema, errorToMessage, isRetryable?(response, error?)}): empty/whitespace body → APICallError with message=response.statusText; else parseJSON against errorSchema → APICallError{message: errorToMessage(parsed), data: parsed, isRetryable: isRetryable?.(response, parsed)}; parse failure falls back to statusText variant. createBinaryResponseHandler: response.arrayBuffer() → Uint8Array, throws APICallError('Response body is empty') if body null. createStatusCodeErrorResponseHandler: APICallError with statusText + full text body.

### createEventSourceResponseHandler + parseJsonEventStream ([DONE] handling)
response-handler.ts line 101: throws EmptyResponseBodyError if response.body == null, else value = parseJsonEventStream({stream: response.body, schema: chunkSchema}) returning ReadableStream<ParseResult<T>>. parse-json-event-stream.ts: stream.pipeThrough(TextDecoderStream).pipeThrough(new EventSourceParserStream()).pipeThrough(TransformStream{ transform({data}, controller){ if (data === '[DONE]') return; controller.enqueue(await safeParseJSON({text: data, schema})); }}). Note: the SSE `event:` field and `id:` are DISCARDED at this layer — only `data` is parsed; schema validation failure does not error the stream, it enqueues {success:false, error, rawValue}. There is NO createJsonStreamResponseHandler (NDJSON) in this version — grep across all packages returns nothing; SSE is the only streaming wire format handled here.

### EventSourceParserStream / SSE parsing algorithm (eventsource-parser 3.1.0, dep ^3.0.8)
createParser({onEvent, onError, onRetry, onComment, maxBufferSize}) returns {feed(chunk:string), reset({consume?})}. State: pendingFragments[] (partial-line buffer joined lazily to avoid O(N^2)), id, data, dataLines, eventType, isFirstChunk (UTF-8 BOM 0xEF 0xBB 0xBF stripped from first chunk only), terminated flag after maxBufferSize overflow. processLines: fast path when chunk has no '\r' — scan indexOf('\n'); blank line dispatches event iff dataLines>0 as {id, event: eventType, data} then resets fields; `data:`/`data: ` (single optional space skipped) appends value, multi-line data joined with '\n'; single-line-event fast path (data line followed immediately by LF dispatches without buffering); `event:` sets eventType (empty → undefined); `id:` sets id unless value contains '\0'; `retry:` must be all ASCII digits → onRetry(parseInt) else ParseError('invalid-retry'); lines starting ':' are comments; unknown fields → ParseError('unknown-field'). Slow path handles \r, \n, \r\n terminators, defers a trailing '\r' at chunk end (could be half of CRLF). Trailing partial line returned and buffered for next feed. EventSourceParserStream is TransformStream<string, EventSourceMessage{event?: string, id?: string, data: string}>; parse errors ignored by default, 'terminate' or buffer overflow errors the stream. Field names compared case-sensitively, no 'message' default event type.

### retryWithExponentialBackoff — lives in provider-utils; header-aware wrapper in ai package
provider-utils/src/retry-with-exponential-backoff.ts: retryWithExponentialBackoff({maxRetries=2, initialDelayInMs=2000, backoffFactor=2, abortSignal, shouldRetry(error)→bool|Promise, getDelayInMs({error, exponentialBackoffDelay})=identity, createRetryError({message, reason:'maxRetriesExceeded'|'errorNotRetryable', errors})}) → RetryFunction = <O>(fn:()=>PromiseLike<O>)=>PromiseLike<O>. Recursive: on error — abort rethrown as-is; maxRetries===0 rethrow unwrapped; tryNumber>maxRetries → createRetryError(maxRetriesExceeded); if shouldRetry → delay(delayInMs, {abortSignal}) (abortable setTimeout that rejects DOMException('AbortError')) then recurse with delayInMs*backoffFactor; non-retryable on first try rethrown unwrapped, otherwise createRetryError(errorNotRetryable). packages/ai/src/util/retry-with-exponential-backoff.ts adds retryWithExponentialBackoffRespectingRetryHeaders: shouldRetry = APICallError.isInstance && isRetryable===true (or GatewayError); getDelayInMs parses 'retry-after-ms' (float ms, OpenAI) with priority over 'retry-after' (float seconds ×1000, else Date.parse(v)-now); accepted only if 0<=ms && (ms<60000 || ms<exponentialBackoffDelay), else exponential value. prepare-retries.ts: default maxRetries 2, validates integer>=0, RetryError from 'ai'.

### secureJsonParse + parseJSON/safeParseJSON
secure-json-parse.ts (fastify port): JSON.parse then, if two regexes (obfuscation-resistant char-class patterns matching "__proto__": and "constructor": including \uXXXX-escaped spellings) match the RAW TEXT, BFS-walk parsed object and throw SyntaxError if any node has own '__proto__' property or a 'constructor' own-property that is an object with own 'prototype'. parse-json.ts: parseJSON({text, schema?}) → secureJsonParse then validateTypes; wraps errors in JSONParseError unless already JSONParseError/TypeValidationError. safeParseJSON returns ParseResult<T> = {success:true, value, rawValue} | {success:false, error: JSONParseError|TypeValidationError, rawValue}. isParsableJson(input) = try/catch probe. Both async (schema.validate may be async).

### fixJson — partial-JSON repair (packages/ai/src/util/fix-json.ts, used by parsePartialJson)
parsePartialJson(jsonText) → {value, state:'undefined-input'|'successful-parse'|'repaired-parse'|'failed-parse'}: try safeParseJSON raw; on failure safeParseJSON(fixJson(text)). fixJson: single linear scan with explicit state stack (states: ROOT, FINISH, INSIDE_STRING, INSIDE_STRING_ESCAPE, INSIDE_STRING_UNICODE_ESCAPE, INSIDE_LITERAL, INSIDE_NUMBER, INSIDE_OBJECT_START/KEY/AFTER_KEY/BEFORE_VALUE/AFTER_VALUE/AFTER_COMMA, INSIDE_ARRAY_START/AFTER_VALUE/AFTER_COMMA). Tracks lastValidIndex (last char that can legally end truncated output — e.g. inside a string every char updates it, but an unterminated '\' escape or <4 hex digits of \uXXXX do not; '-' starting a number does not until a digit; trailing '.', 'e', 'E' in numbers excluded; partial literals tracked via literalStart and prefix-of 'true'/'false'/'null' checks). Output = input.slice(0, lastValidIndex+1) then close the stack top-down: INSIDE_STRING appends '"', object states append '}', array states append ']', INSIDE_LITERAL completes the literal ('tr' → 'true'). Result: trailing commas/partial keys/values-in-progress are dropped, structures closed. Assumes input is a prefix of valid JSON (invalid JSON not covered).

### Size limits, socket hygiene, SSRF guard
readResponseWithSizeLimit: DEFAULT_MAX_DOWNLOAD_SIZE = 2 GiB; early-rejects on Content-Length (cancels body first to return connection to pool), else incremental reader accumulating chunks with running total, DownloadError on overflow; finally reader.cancel()+releaseLock; concatenates into one Uint8Array. cancelResponseBody(response): response.body?.cancel() with errors swallowed — called everywhere a response is rejected unread to avoid fd leaks. fetchWithValidatedRedirects: manual redirect following (redirect:'manual'), validateDownloadUrl on every hop, max 10 redirects, 3xx+Location → cancel body, resolve URL relative to current; opaqueredirect: browser refetches with follow, non-browser fails closed. validateDownloadUrl: allows data:, only http/https otherwise; blocks localhost/.local/.localhost (trailing-dot stripped), private IPv4 ranges (0/8, 10/8, 100.64/10, 127/8, 169.254/16, 172.16/12, 192.0.0/24, 192.168/16, 198.18/15, >=240), IPv6 loopback/unspecified/ULA fc00::/7/link-local fe80::/10/site-local fec0::/10/multicast ff00::/8 plus IPv4-embedded forms (::/96, ::ffff:0:0/96, NAT64 64:ff9b::/96 and 64:ff9b:1::/48) checked against the IPv4 rules; unparseable IPv6 fails closed. String-only checks, no DNS resolution.

## Port implications
The whole layer maps cleanly onto std.http.Client + std.Io with a few deliberate design decisions. (1) ResponseHandler<T> is the central polymorphism point: in Zig make it a tagged union or vtable struct `fn handle(ctx, *Response, url, request_body) HandlerError!HandlerResult(T)` — comptime-generic over T works for the pure-Zig API, but the C FFI needs a type-erased variant returning an owned buffer + parse status. (2) AbortSignal has no direct equivalent; Zig 0.16's std.Io async model means cancellation should be modeled as io.Cancelable/cancellation token threaded through fetch, delay, and retry — the TS code checks aborts at exactly three points (fetch signal, delay, retry rethrow), so a single `*std.atomic.Value(bool)` or Io cancel token checked at those points is faithful. (3) SSE parsing is the hardest 1:1 piece: eventsource-parser's string-based fragment buffering should become a byte-oriented incremental parser over std.Io.Reader — keep the exact spec semantics (BOM strip on first chunk, \r/\n/\r\n with deferred trailing CR, multi-line data joined with \n, id NUL rejection, retry digit validation, dispatch-on-blank-line only when dataLines>0), but replace the JS fast paths with a simple line scanner; the TransformStream pipeline (TextDecoder→SSE→JSON) collapses into one pull-based iterator `next() !?ParseResult(T)` that drops '[DONE]'. UTF-8 validation replaces TextDecoderStream. (4) fixJson is a perfect Zig port target: explicit state enum + ArrayList stack, returns slice length + closer string — allocate output as input.len + stack depth upper bound. (5) safeParseJSON's ParseResult tagged union maps to Zig error unions poorly (need the error payload), so use `union(enum){success: struct{value: T, raw: std.json.Value}, failure: ...}`. secureJsonParse's proto-poisoning guard is unnecessary in Zig (no prototype chain) — document as intentionally omitted, but keep std.json parsing strict. (6) Schema abstraction: replace Zod with comptime std.json parsing into typed structs plus an optional runtime JSONSchema value for the wire; Validator = optional function pointer, matching Schema.validate?. (7) Retry: two layers as in TS — generic engine in the provider-utils-equivalent module, Retry-After header awareness in the ai layer; delay must be an Io-aware cancellable sleep. Iterative loop instead of recursion; errors array = bounded ArrayList of captured error info (Zig errors carry no payload — need an ErrorInfo struct mirroring APICallError fields: status, headers, body, is_retryable). (8) Headers: plain StringHashMap with lowercased keys reproduces normalizeHeaders/combineHeaders; user-agent suffix is trivial. (9) Size-limited body read maps to a counting Reader wrapper; cancelResponseBody ≈ closing/draining the connection — std.http.Client connection pooling makes the socket-leak concern real, so mirror the cancel-on-reject discipline. (10) FetchFunction injection: define a `HttpTransport` interface (vtable) so tests and the C FFI can substitute transports, exactly as fetch is injected everywhere. Note for fidelity tracking: createJsonStreamResponseHandler (NDJSON) no longer exists upstream; retryWithExponentialBackoff lives in provider-utils with the header-respecting wrapper in packages/ai.
