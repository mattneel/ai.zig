# zig-ffi-export

## Summary
Researched Zig 0.16 C ABI export mechanics against stdlib source, ziglang tree, release notes, and ziex corpus, then validated everything empirically: built a prototype C ABI library (opaque runtime handle over std.Io.Threaded, enum(c_int) status codes, ptr+len strings, callee-allocated out-params with free fn, callconv(.c) callback+user_data, push- and pull-based streaming over Io.concurrent/Io.Queue) with zig 0.16.0 via b.addLibrary into both libai.a and versioned libai.so.0.1.0 with an installed hand-written header, then exercised it from Python ctypes and a C program compiled with zig cc. Confirmed: -femit-h is dead in 0.16 (CLI fatals; no header-gen code exists in the compiler) so headers must be hand-written; addSharedLibrary/addStaticLibrary are replaced by addLibrary(.{.linkage=...}); export fn implies the target C callconv and rejects slices/arrays/error-unions/inferred-tag enums (0.16 newly forbids implicit backing types in extern contexts); Io is a thread-safe {userdata,vtable} fat pointer where Io.Threaded works from arbitrary foreign threads (blocking, no loop to drive) while the fiber-based Evented/Uring implementations are pinned to their init thread; Future.cancel from Python interrupted a sleeping producer in 0.2 ms; environment variables are non-global in 0.16 so a library must read libc environ or take explicit config; Threaded.init installs process-global SIGIO/SIGPIPE handlers, a real concern when embedding in Python.

## Key findings
### export fn / callconv rules (0.16)
`export fn` gets the target C calling convention: std.builtin.CallingConvention is now `union(enum(u8))` (builtin.zig:125) with per-arch payload variants; `.c` is `pub const c = builtin.target.cCallingConvention().?`. Allowed param/return types are extern-compatible only — compile errors verified in test/cases/compile_errors: slices ('slices have no guaranteed in-memory representation'), by-value arrays, comptime/generic params, non-extern struct/union/enum all rejected with "parameter of type 'X' not allowed in function with calling convention 'x86_64_sysv'". Alias exports: `export const foo_exported = foo_internal;` (internal fn must be callconv(.c)) or `@export(&f, .{ .name, .linkage = .strong, .section = null, .visibility = .default })` (ExportOptions builtin.zig:1090). Import side: `@extern(*const fn () callconv(.c) i32, .{ .name = "sym" })`. GlobalLinkage = {internal,strong,weak,link_once}; SymbolVisibility = {default,hidden,protected}.

### 0.16 breaking change: implicit backing types forbidden in extern contexts
Release notes 'Forbid Enum and Packed Types with Implicit Backing Types in Extern Contexts' (proposal #24714): `enum {a,b}`, `packed struct {a: u4, b: u4}`, `packed union` without explicit backing int are no longer valid extern types — 'error: unable to export type ...; integer tag type of enum is inferred'. Fix: `enum(u8)`, `enum(c_int)`, `packed struct(u8)`, `packed union(u8)`. Additionally enum tag ints must be 0 or power-of-two bits: `enum(u31)` in an extern struct fails with 'only integers with 0 or power of two bits are extern compatible'.

### -femit-h is dead; headers are hand-written
src/main.zig:1547 hard-fatals on `-femit-h` ('currently broken, see ziglang/zig#9698'); `-femit-h=path` parses but NO header-generation code exists anywhere in the 0.16 compiler (emit_h is plumbed into Compilation options and consumed nowhere; test/gen_h.zig is orphaned, unreferenced by tests.zig). Build API remnants: Step.Compile.getEmittedH (Compile.zig:869) appends -femit-h (line 1411) which would fatal; InstallArtifact's emitted_h wiring is commented out (InstallArtifact.zig:124). Correct pattern: hand-write include/ai.h, ship with `lib.installHeader(b.path("include/ai.h"), "ai.h")` or `installHeadersDirectory(path, "prefix", .{})`; dependents via linkLibrary get the include tree automatically (getEmittedIncludeTree), installed copies land in zig-out/include.

### build.zig 0.16: one addLibrary for static AND shared
addSharedLibrary/addStaticLibrary no longer exist. `b.addLibrary(.{ .linkage: std.builtin.LinkMode = .static (or .dynamic), .name, .root_module: *Module, .version: ?SemanticVersion, .win32_module_definition: ?LazyPath, ... })` (Build.zig:842). Modules: `b.createModule(.{ .root_source_file, .target, .optimize, .link_libc: ?bool, .pic: ?bool, .single_threaded, .strip, .sanitize_c, ... })` (Module.zig:216). Verified: the SAME *Module instance can back both a .dynamic and a .static Compile step; `.version = .{.major=0,.minor=1,.patch=0}` yields libai.so.0.1.0 + libai.so.0 + libai.so symlinks. `bundle_compiler_rt: ?bool` defaults true only for exe/dylib (Compile.zig:699) — set `static_lib.bundle_compiler_rt = true` so C linkers don't miss compiler-rt. Other relevant Compile fields: version_script, rdynamic, dll_export_fns, force_undefined_symbols, win32_manifest. Debug builds export weak `__ubsan_handle_*` symbols (bundle_ubsan_rt to control). Note: anyzig requires build.zig.zon with minimum_zig_version + fingerprint before `zig build` runs.

### Validated C ABI surface patterns (built + run on 0.16.0)
Prototype at scratchpad/cabi compiled clean and passed Python ctypes + C consumer tests. Patterns: (1) opaque handle `pub const ai_runtime = opaque {};` returned via out-param `export fn ai_runtime_create(out: *?*ai_runtime) AiStatus`, internal recovery `@ptrCast(@alignCast(h))` to `*Runtime` (heap struct holding DebugAllocator + Io.Threaded, created with std.heap.c_allocator); (2) status codes `pub const AiStatus = enum(c_int) { ok = 0, ... unknown = -1 }` used directly as export return type; `export fn ai_status_name(s: AiStatus) [*:0]const u8 { return @tagName(s); }` works ([: 0]const u8 coerces); (3) strings in as `text_ptr: [*]const u8, text_len: usize`; outputs as callee-allocated `out_ptr: *?[*]u8, out_len: *usize` + paired `ai_buf_free(ptr,len)` freeing with the same c_allocator; (4) callback type `*const fn (user_data: ?*anyopaque, chunk_ptr: [*]const u8, chunk_len: usize) callconv(.c) void` + `?*anyopaque` user_data — invoked from an Io.Threaded pool thread into Python; ctypes CFUNCTYPE handled cross-thread GIL acquisition transparently. Precedent corpus: ziex vendor/zig-tree-sitter (opaque {} handles, extern struct Logger{payload:?*anyopaque, log:?*const fn(...) callconv(.c)}), ziex src/runtime/client exports use ptr+len pairs.

### std.Io shape and thread-safety for FFI
Io = `{ userdata: ?*anyopaque, vtable: *const VTable }` (Io.zig:25). All VTable entries (async, concurrent, await, cancel, groupAsync..., operate, dir*/file*/net*) are documented 'Thread-safe'. `Io.Threaded.init(gpa: Allocator, InitOptions) Threaded` — infallible value init; gpa must be threadsafe but only used by async/concurrent (Allocator.failing legal otherwise); `t.io() Io` (Threaded.zig:1806); `t.deinit()` joins pool. InitOptions: stack_size, async_limit (default cpus-1), concurrent_limit, argv0, environ, disable_memory_mapping. CRITICAL for embedding: on POSIX, init installs process-global no-op SIGIO and SIGPIPE handlers (Threaded.zig:~1660) used to interrupt blocking syscalls for cancellation; deinit restores old handlers — visible to Python hosts. Foreign-thread calls are safe: `threadlocal var current: ?*Thread` is null-tolerant (currentId() falls back to getCurrentId; checkCancel returns void for unregistered threads). `Io.Threaded.global_single_threaded: *Threaded` exists (no concurrency/cancellation).

### Blocking C API over async core: recommended architecture (validated)
Threaded has NO event loop to drive — every Io op blocks the calling thread; async/concurrent lazily spawn pool threads. So the blocking C wrapper is trivial: a long-lived opaque runtime handle owning Io.Threaded; each exported fn calls the Zig core passing rt.threaded.io() and blocks (colorblind functions). Per-call Threaded instantiation is possible (init is cheap, infallible) but pays sigaction syscalls + thread-pool warmup each call and toggles global signal handlers — long-lived handle is right. By contrast Io.Evented (= Uring on Linux, Kqueue on BSDs, Dispatch on Darwin, fiber-based; `pub const Evented` in Io.zig:30) is PINNED to the init thread: Uring.init(ev,*,allocator,options) captures caller as main fiber and sets threadlocal Thread.self; deinit asserts `Thread.current().currentFiber() == main_fiber`. C callers cannot call an Evented Io from arbitrary threads — using it under FFI would require the library to own a dedicated runtime thread and marshal requests. Default the FFI runtime to Threaded.

### Streaming + cancellation across the boundary (validated)
Push model: `var fut = io.concurrent(worker, .{io, cb, user_data}) catch return .unknown; fut.await(io);` — `Io.concurrent(io, function, args: std.meta.ArgsTuple) ConcurrentError!Future(R)` (Io.zig:2365; error.ConcurrencyUnavailable), Future(R) has `await(f,io) R` and `cancel(f,io) R`, both idempotent, NOT threadsafe per-future. Pull model for Python iterators: `Io.Queue(Elem)` (Io.zig:2184) bounded on caller buffer — `.init(buffer)`, `putOne(io, elem) (QueueClosedError||Cancelable)!void`, `getOne(io) !Elem`, `close(io)`, plus putUncancelable variants. Validated: producer task pushing heap chunks into Queue(Chunk{ptr,len}); Python pulled 5 of 100 via blocking ai_stream_next, then ai_stream_close ran future.cancel(io) from the Python thread — producer blocked in io.sleep was interrupted in 0.2 ms (signal-based cancel), queue drained, no leaks. Cancellation semantics: cancel sets a flag; next 'cancelation point' (any Io call returning error.Canceled) throws once. `io.sleep(Duration, Clock)` with `Duration.fromMilliseconds(i64)`, Clock = {real, awake, boot, cpu_process, cpu_thread}. Sync primitives are all Io-based now: Io.Mutex (extern struct), Io.Event enum(u32), Io.Group, Io.Queue, futexWait/futexWake vtable entries; std.Thread.{Mutex,Condition,WaitGroup,...} migrated to std.Io.* per release notes.

### Environment variables are non-global in 0.16 (library implication)
Release notes 'Environment Variables and Process Arguments Become Non-Global': env is only handed to `pub fn main(init: std.process.Init)` ('Juicy Main'); std.os.environ is gone. A shared library has no Init, so for env-based config (ANTHROPIC_API_KEY etc.) either (a) link libc and build `std.process.Environ` from `pub extern "c" var environ: [*:null]?[*:0]u8` (std/c.zig:10633) with PosixBlock{.slice=...}, or (b) require explicit config through the C API (cleaner for FFI). `Threaded.InitOptions.environ: process.Environ = .empty` only affects processSpawn/fileIsTty/executablePath; Environ.Block is GlobalBlock (use_global flag) on Windows, PosixBlock ([:null]?[*:0]const u8 slice) elsewhere.

### DLL/start.zig and export analysis mechanics
start.zig comptime: for `output_mode == .Lib and link_mode == .dynamic` on Windows it auto-exports `_DllMainCRTStartup` unless root declares it; it also does `_ = root;` which forces analysis of the root module so top-level `export fn`s in the root source file are emitted. If C API exports live in sub-files of the root module, root must reference them (`comptime { _ = @import("capi_stream.zig"); }`) since usingnamespace no longer exists. .def files via LibraryOptions.win32_module_definition; ELF symbol filtering via compile.version_script.

### Header round-trip testing and comptime C-type generation
@cImport is deprecated in 0.16 → build-system translate-c: `const tc = b.addTranslateC(.{ .root_source_file = b.path("include/ai.h"), .target, .optimize }); mod.addImport("c", tc.createModule())` — ideal for an ABI-lock test that imports the hand-written header and comptime-asserts signatures/enum values match the Zig exports. Also new in 0.16: @Type is replaced by @Fn/@Struct/@Union/@Enum builtins (SoA-style args, e.g. `@Fn(param_types, &@splat(.{}), Ret, .{ .@"callconv" = .c })`, `@Struct(.@"extern", null, names, types, attrs)`) enabling comptime generation of extern mirror structs from Zig SDK types.

### Allocators for the FFI boundary
std.heap.c_allocator (heap.zig:139, requires link_libc), smp_allocator (heap.zig:353), page_allocator, wasm_allocator all present in 0.16. Pattern validated: buffers crossing the boundary allocated with c_allocator so a paired exported free fn works regardless of internal allocator choice; internal runtime state can use DebugAllocator (thread_safe defaults to !single_threaded, debug_allocator.zig:127) for leak detection in debug builds. Zig panics abort the process and must never unwind into C — every exported fn must be total: `catch return .status_code`.

## Port implications
Design the FFI as a thin hand-written layer: (1) One long-lived opaque `ai_runtime` handle wrapping {threadsafe gpa, Io.Threaded}; every exported fn is a blocking call passing rt.threaded.io() into the colorblind Zig core — no event loop for C to drive, and per-call Threaded init is wasteful (sigaction + pool warmup). Do NOT use Evented/Uring under FFI without a dedicated marshaling thread (fiber runtime is init-thread-pinned). (2) The C header must be hand-written and shipped via installHeader — emit-h is broken; add an ABI-lock test using b.addTranslateC on the shipped header comparing against the export declarations. (3) All extern types need explicit backing ints (enum(c_int), packed struct(u32)); no slices/error-unions across the boundary — ptr+len pairs, enum(c_int) status codes with a Zig error->status switch, @tagName for diagnostics. (4) Streaming: push model (callback + user_data from io.concurrent worker, callconv(.c)) and pull model (opaque stream handle over Io.Queue(Chunk) with blocking next() and cancel via Future.cancel) both validated from Python; pull model is the natural fit for Python/Rust iterators, and cancellation latency is sub-millisecond. Buffers crossing the boundary should come from c_allocator with a paired free export. (5) build.zig: single shared *Module feeding addLibrary twice (.dynamic with .version for soname, .static with bundle_compiler_rt=true); link_libc=true, pic=true. (6) Document the embedding caveats: Threaded.init replaces SIGIO/SIGPIPE dispositions process-wide; env vars are non-global in 0.16 so API keys/proxy config must be explicit C API parameters (or read libc environ when libc is linked); panics abort the host process so every export must be total. Hard parts: keeping the hand-written header, ctypes/bindgen decls, and Zig exports in sync (mitigate with the translate-c ABI test), and lifetime rules for chunks handed to callbacks vs. pulled chunks (callee-owned until explicit free).
