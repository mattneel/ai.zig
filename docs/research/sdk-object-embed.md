# sdk-object-embed

## Summary
Deep-read of generate-object/, embed/, rerank/ in packages/ai/src plus the schema abstraction in packages/provider-utils. generateObject is a single-step (stepNumber always 0) flow: validate input combo, build an OutputStrategy (object/array/enum/no-schema), get its JSON Schema, call model.doGenerate with responseFormat {type:'json', schema, name, description} under a retry wrapper, extract text content, then parse (secureJsonParse) + validateFinalResult, with a one-shot experimental_repairText fallback; all failures surface as NoObjectGeneratedError carrying text/response/usage/finishReason. streamObject returns a result object synchronously; a detached async task pipes LanguageModelV4StreamPart through two TransformStreams into a stitchable stream, accumulating text, running parsePartialJson (safeParseJSON, then fixJson repair — a single-pass state-machine scanner that truncates to last-valid index and appends closers) on every delta, deduping via isDeepEqualData, and emitting ObjectStreamPart {object|text-delta|error|finish}; seven DelayedPromises expose object/usage/warnings/request/response/finishReason/providerMetadata. Array mode wraps element schema in {elements:[...]}, validates all-but-last element per delta, and synthesizes array-syntax textDeltas; enum mode wraps in {result: string} with prefix-matching partial validation. The schema abstraction (Schema<T>) needs only two capabilities: a (possibly async) JSON Schema draft-07 document and an optional validate(unknown)→{success,value}|{success,error}. embedMany batches by model.maxEmbeddingsPerCall, then runs waves of Promise.all sized maxParallelCalls (or 1 if !supportsParallelCalls), summing tokens and merging providerMetadata. rerank maps provider ranking {index, relevanceScore} back onto original documents.

## Key findings
### OutputStrategy interface (output-strategy.ts)
OutputStrategy<PARTIAL, RESULT, ELEMENT_STREAM> = { type: 'object'|'array'|'enum'|'no-schema'; jsonSchema(): Promise<JSONSchema7|undefined>; validatePartialResult({value: JSONValue, textDelta: string, isFirstDelta: bool, isFinalDelta: bool, latestObject: PARTIAL|undefined}) -> Promise<ValidationResult<{partial: PARTIAL, textDelta: string}>>; validateFinalResult(value: JSONValue|undefined, ctx {text, response, usage}) -> Promise<ValidationResult<RESULT>>; createElementStream(ReadableStream<ObjectStreamPart<PARTIAL>>) -> ELEMENT_STREAM (throws UnsupportedFunctionalityError except in array mode) }. getOutputStrategy dispatches on output string; object/array call asSchema(schema), enum takes string[] values.

### object strategy
jsonSchema = schema.jsonSchema (awaited). validatePartialResult is a no-op passthrough (partial NOT validated, cast to DeepPartial<OBJECT>). validateFinalResult = safeValidateTypes({value, schema}) which uses schema.validate if present, else passes value through unvalidated.

### array strategy
jsonSchema wraps element schema: strips $schema from item schema, returns {$schema: draft-07, type:'object', properties:{elements:{type:'array', items:itemSchema}}, required:['elements'], additionalProperties:false} because LLMs cannot emit top-level arrays. validatePartialResult: requires isJSONObject(value) && isJSONArray(value.elements) else TypeValidationError; validates each element via safeValidateTypes but SKIPS the last element unless isFinalDelta (it is presumed incomplete); any earlier element failure aborts. textDelta synthesis: '[' if isFirstDelta, ',' if publishedElementCount>0, then JSON.stringify of only elements[publishedElementCount..], ']' if isFinalDelta — so textStream emits valid top-level array JSON even though wire format is the wrapper object. createElementStream: TransformStream keeping publishedElements counter, on 'object' chunks enqueues array[publishedElements..] one by one; text-delta/finish/error suppressed.

### enum strategy
jsonSchema wraps: {type:'object', properties:{result:{type:'string', enum: enumValues}}, required:['result'], additionalProperties:false}. validatePartialResult: value must be object with string .result; filters enumValues by startsWith(result); fails if result empty or no prefix candidates; partial = result if >1 candidates else the single full candidate (early completion). validateFinalResult: membership check enumValues.includes(result). Enum values validated as strings at input-validation time.

### no-schema strategy
jsonSchema() -> undefined (no responseFormat schema sent). validatePartialResult always succeeds with raw JSONValue. validateFinalResult fails only when value === undefined, producing NoObjectGeneratedError('response did not match schema') with text/response/usage/finishReason context. Element streams unsupported.

### validateObjectGenerationInput
Cross-checks output mode vs args, throwing InvalidArgumentError: no-schema forbids schema/schemaName/schemaDescription/enumValues; object and array require schema, forbid enumValues; enum forbids schema/schemaName/schemaDescription, requires enumValues, every value must be typeof 'string'.

### generateObject control flow (generate-object.ts)
1) resolveLanguageModel (string -> globalProvider/gateway lookup); 2) validateObjectGenerationInput; 3) prepareRetries -> {maxRetries (default 2), retry}; 4) getOutputStrategy; 5) jsonSchema = await strategy.jsonSchema(); callId = generateId (prefix 'aiobj', size 24); 6) notify onStart; 7) standardizePrompt + convertToLanguageModelPrompt; 8) notify onStepStart (stepNumber always 0); 9) retry(() => model.doGenerate({responseFormat:{type:'json', schema, name: schemaName, description: schemaDescription}, ...callSettings, prompt, providerOptions, abortSignal, headers: withUserAgentSuffix(headers, 'ai/VERSION')})); 10) text = extractTextContent(result.content); undefined -> throw NoObjectGeneratedError('model did not return a response'); 11) notify onStepEnd with objectText=text; 12) object = parseAndValidateObjectResultWithRepair; 13) notify onFinish; return DefaultGenerateObjectResult{object, reasoning, finishReason, usage, warnings, request, response, providerMetadata, toJsonResponse()}. Catch: telemetry onError then throw wrapGatewayError(error).

### parse+validate+repair pipeline (parse-and-validate-object-result.ts)
parseAndValidateObjectResult: safeParseJSON({text}) fail -> NoObjectGeneratedError('could not parse the response', cause=JSONParseError); strategy.validateFinalResult fail -> NoObjectGeneratedError('response did not match schema', cause=TypeValidationError); both carry {text, response, usage, finishReason}. WithRepair wrapper: catches NoObjectGeneratedError whose cause is JSONParseError or TypeValidationError, calls repairText({text, error}) (type RepairTextFunction = (opts:{text, error: JSONParseError|TypeValidationError}) => Promise<string|null>); null -> rethrow original; else re-run parse+validate exactly once on repaired text (second failure propagates).

### streamObject architecture (stream-object.ts)
streamObject returns DefaultStreamObjectResult synchronously. Fields: 7 DelayedPromise instances (_object, _usage, _providerMetadata, _warnings, _request, _response, _finishReason) + baseStream. Constructor: creates stitchableStream<ObjectStreamPart<PARTIAL>>; baseStream = stitchable.stream.pipeThrough(eventProcessor) where eventProcessor re-emits every chunk and calls onError({error: wrapGatewayError(chunk.error)}) on 'error' chunks (default onError = console.error). Then an async IIFE: awaits jsonSchema, notify onStart/onStepStart, standardizePrompt/convert, retry(() => model.doStream(callOptions)) with callOptions including responseFormat json + includeRawChunks:false; on IIFE rejection, catch adds a one-chunk error stream to the stitchable; finally stitchable.close().

### streamObject transform pipeline
Stage 1 Transformer<LanguageModelV4StreamPart, ObjectStreamInputPart>: 'text-delta' -> enqueue chunk.delta (plain string); response-metadata/finish/error/stream-start passed through; everything else dropped. Stage 2 TransformStream<string|ObjectStreamInputPart, ObjectStreamPart<PARTIAL>> with closure state: accumulatedText, textDelta, latestObjectJson, latestObject, isFirstChunk (msToFirstChunk = now()-start on first), isFirstDelta, fullResponse{id,timestamp,modelId}. String chunk: accumulatedText += chunk; textDelta += chunk; {value, state} = await parsePartialJson(accumulatedText); if value !== undefined && !isDeepEqualData(latestObjectJson, value): run strategy.validatePartialResult({value, textDelta, latestObject, isFirstDelta, isFinalDelta: state==='successful-parse'}); if success && partial deep-differs from latestObject: enqueue {type:'object', object: partial} then {type:'text-delta', textDelta: validationResult.value.textDelta}, reset textDelta='', isFirstDelta=false. 'stream-start' captures warnings. 'response-metadata' merges into fullResponse. 'finish': flush residual textDelta chunk, set finishReason=chunk.finishReason.unified, usage=asLanguageModelUsage(chunk.usage), enqueue finish part {finishReason, usage, response: fullResponse, providerMetadata}, resolve _usage/_providerMetadata/_warnings/_response/_finishReason, then try parseAndValidateObjectResultWithRepair(accumulatedText,...) -> _object.resolve(object) or _object.reject(e) with error stored for onFinish. flush(): notify onStepEnd (objectText=accumulatedText, msToFirstChunk) then onFinish (object may be undefined, error set); notify errors are enqueued as error chunks.

### StreamObjectResult surface (stream-object-result.ts)
Interface: warnings/usage/providerMetadata/request/response/finishReason/object as Promises; partialObjectStream: AsyncIterableStream<PARTIAL> (filters 'object' chunks); elementStream: ELEMENT_STREAM (delegates to strategy.createElementStream(baseStream)); textStream: AsyncIterableStream<string> (filters 'text-delta'; concatenation is valid JSON at end); fullStream: AsyncIterableStream<ObjectStreamPart<PARTIAL>>; pipeTextStreamToResponse(ServerResponse, init); toTextStreamResponse(init) -> Response. ObjectStreamPart<PARTIAL> = {type:'object', object} | {type:'text-delta', textDelta} | {type:'error', error: unknown} | {type:'finish', finishReason, usage, response, providerMetadata?}. Note each stream getter pipes baseStream through a new TransformStream — baseStream is single-consumer; accessing two stream getters competes for the same source. Type-level: PARTIAL is string for enum, RESULT (elements array) for array, DeepPartial<RESULT> for object.

### parsePartialJson + fixJson (util/parse-partial-json.ts, util/fix-json.ts)
parsePartialJson(text) -> {value: JSONValue|undefined, state: 'undefined-input'|'successful-parse'|'repaired-parse'|'failed-parse'}: try safeParseJSON; else safeParseJSON(fixJson(text)); else failed. fixJson: single linear scan with explicit state stack (17 states: ROOT, FINISH, INSIDE_STRING, INSIDE_STRING_ESCAPE, INSIDE_STRING_UNICODE_ESCAPE (counts 4 hex digits), INSIDE_LITERAL, INSIDE_NUMBER, INSIDE_OBJECT_START/KEY/AFTER_KEY/BEFORE_VALUE/AFTER_VALUE/AFTER_COMMA, INSIDE_ARRAY_START/AFTER_VALUE/AFTER_COMMA). Tracks lastValidIndex; result = input[0..lastValidIndex+1] then walks stack top-down appending closers: '"' for open string, '}' for any object state, ']' for any array state, and completes partial true/false/null literals. Effect: dangling keys, colons, trailing commas, incomplete escapes, incomplete numbers (trailing e/E/./-) are truncated; open scopes closed. Pure string algorithm, no allocator tricks needed beyond a state stack + output buffer.

### isDeepEqualData (util/is-deep-equal-data.ts)
Recursive structural equality over parsed JSON: strict ===, null checks, constructor equality, Date getTime special-case, array length+elementwise, object key-set + per-key recursion. Called up to twice per stream delta (on raw JSON and on validated partial) — hot path for streaming; a Zig port over std.json.Value should consider this cost.

### Schema<T> core abstraction (provider-utils/schema.ts)
Schema<OBJECT> = { [Symbol.for('vercel.ai.schema')]: true; _type: OBJECT (phantom, inference only); validate?: (value: unknown) => ValidationResult<OBJECT> | PromiseLike<...>; jsonSchema: JSONSchema7 | PromiseLike<JSONSchema7> }. ValidationResult<T> = {success:true, value:T} | {success:false, error:Error}. That is the ENTIRE contract consumed by ai core: (1) a lazily/async-obtainable JSON Schema draft-07 document to send to providers, (2) an OPTIONAL async validator that coerces unknown -> typed value. validate==null means values pass through unvalidated (safeValidateTypes returns success with raw value). jsonSchema() factory accepts a JSONSchema7, a PromiseLike, or a thunk (cached on first get). LazySchema<T> = () => Schema<T> with memoization.

### FlexibleSchema and asSchema normalization
FlexibleSchema<T> = Schema<T> | LazySchema<T> | ZodSchema<T> (zod v3 or v4) | StandardSchema<T> (StandardSchemaV1 & StandardJSONSchemaV1). asSchema(schema): null/undefined -> jsonSchema({type:'object', properties:{}, additionalProperties:false}); isSchema (symbol marker + jsonSchema + validate keys) -> passthrough; '~standard' in schema -> vendor==='zod' ? zodSchema() : standardSchema(); else invoke as LazySchema thunk. standardSchema uses standard['~standard'].jsonSchema.input({target:'draft-07'}) and standard['~standard'].validate(value) mapping issues -> TypeValidationError. zod4Schema uses z4.toJSONSchema(schema, {target:'draft-7', io:'input', reused: useReferences?'ref':'inline'}); zod3 uses vendored zod3ToJsonSchema with $refStrategy 'root'|'none'. Both post-process via addAdditionalPropertiesToJsonSchema which recursively forces additionalProperties:false on all object schemas (walking properties/items/anyOf/allOf/oneOf/definitions) because e.g. OpenAI structured outputs reject additionalProperties:true. InferSchema<S> and DeepPartial<T> are pure type-level machinery.

### JSON parse layer (provider-utils/parse-json.ts, secure-json-parse.ts)
safeParseJSON({text, schema?}) -> ParseResult<T> = {success:true, value, rawValue} | {success:false, error: JSONParseError|TypeValidationError, rawValue}. Uses secureJsonParse: JSON.parse then rejects payloads containing __proto__ or constructor.prototype keys (prototype-pollution defense — JS-specific, moot in Zig). validateTypes/safeValidateTypes wrap schema.validate with TypeValidationError.wrap. parseJSON is the throwing variant. isParsableJson helper. All async because schema.validate may be async.

### embed control flow (embed/embed.ts)
embed({model, value: string, maxRetries?, abortSignal?, headers?, providerOptions?, telemetry?, onStart?, onEnd?}) -> Promise<EmbedResult>. resolveEmbeddingModel; prepareRetries (default 2); headers get user-agent suffix 'ai/VERSION'; retry wraps a single model.doEmbed({values:[value], abortSignal, headers, providerOptions}); embedding = response.embeddings[0]; usage = response.usage ?? {tokens: NaN}; logWarnings; result {value, embedding: number[], usage: {tokens: number}, warnings: Warning[], providerMetadata?, response?: {headers?, body?}}. Telemetry via runInTracingChannelSpan wrapper + notify (parallel callback invocation, exceptions swallowed per-callback).

### embedMany batching + parallelism (embed/embed-many.ts)
embedMany adds values: string[] and maxParallelCalls (default Infinity). Awaits Promise.all([model.maxEmbeddingsPerCall, model.supportsParallelCalls]) — both may be PromiseLike. Fast path: maxEmbeddingsPerCall null or Infinity -> one retried doEmbed with all values. Chunked path: valueChunks = splitArray(values, maxEmbeddingsPerCall); parallelChunks = splitArray(valueChunks, supportsParallelCalls ? maxParallelCalls : 1); OUTER sequential for-loop over parallelChunks, INNER await Promise.all(parallelChunk.map(chunk => retry(doEmbed(chunk)))). So concurrency is wave-based with a barrier between waves (not a sliding-window semaphore); order of embeddings preserved because chunks are processed/collected in order. Aggregation: embeddings.push(...), warnings concat, responses array (per chunk, possibly undefined), tokens summed (+= usage.tokens, NaN poisons sum), providerMetadata merged per provider-name key with shallow spread (later chunks override). Result: EmbedManyResult {values, embeddings, usage:{tokens}, warnings, providerMetadata?, responses?}.

### EmbeddingModelV4 provider interface
EmbeddingModelV4 = { specificationVersion:'v4'; provider: string; modelId: string; maxEmbeddingsPerCall: PromiseLike<number|undefined>|number|undefined; supportsParallelCalls: PromiseLike<boolean>|boolean; doEmbed(opts: {values: string[], abortSignal?, providerOptions?, headers?}) -> PromiseLike<{embeddings: number[][], usage?: {tokens: number}, providerMetadata?, response?: {headers?, body?}, warnings: Warning[]}> }. Embedding = number[] (EmbeddingModelV4Embedding).

### rerank control flow (rerank/rerank.ts)
rerank<VALUE extends JSONObject|string>({model, documents: VALUE[], query: string, topN?, maxRetries?, abortSignal?, headers?, providerOptions?, telemetry?, onStart?, onEnd?}) -> Promise<RerankResult<VALUE>>. Empty documents: short-circuits with empty ranking, no model call (still fires onStart/onEnd). Document union detection via typeof documents[0] === 'string' -> {type:'text', values} else {type:'object', values}. retry(model.doRerank({documents, query, topN, providerOptions, abortSignal, headers})). Provider returns ranking: [{index, relevanceScore}] sorted desc; core maps to [{originalIndex: r.index, score: r.relevanceScore, document: documents[r.index]}]. RerankResult: originalDocuments, ranking, rerankedDocuments (getter mapping ranking->document), providerMetadata?, response {id?, timestamp, modelId, headers?, body?}. RerankingModelV4 = {specificationVersion:'v4', provider, modelId, doRerank}.

### retry machinery (prepare-retries.ts, retry-with-exponential-backoff*.ts)
prepareRetries validates maxRetries integer >= 0, default 2; returns RetryFunction = <O>(fn: () => PromiseLike<O>) => PromiseLike<O>. Implementation: recursive retryWithExponentialBackoff with initialDelayInMs 2000, backoffFactor 2; never retries AbortError; maxRetries===0 rethrows raw error; shouldRetry = APICallError.isRetryable || GatewayError.isRetryable; delay respects retry-after-ms (float ms) then retry-after (seconds or HTTP date) headers when 0 <= ms and (ms < 60000 or < exponential delay), else exponential; delay() is abortSignal-aware; exhaustion throws RetryError{reason: 'maxRetriesExceeded'|'errorNotRetryable', errors: all attempts}; non-retryable error on first try rethrown unwrapped.

### DelayedPromise + stitchable stream + AsyncIterableStream (concurrency plumbing)
DelayedPromise<T>: stores status {pending|resolved(value)|rejected(error)}; underlying Promise constructed only on .promise access (avoids unhandled rejection when result never awaited) — maps to a Zig future/oneshot with stored result. createStitchableStream<T>: {stream, addStream(inner), close() (graceful: drain then close), terminate() (cancel inners immediately)}; pull-driven: reads inner readers FIFO, blocks on a resolvable promise until a new inner stream is added or closed; inner error -> controller.error + terminate. Used by streamObject so errors thrown before doStream resolves still surface as stream 'error' chunks. AsyncIterableStream<T> = AsyncIterable<T> & ReadableStream<T>; createAsyncIterableStream pipes through identity TransformStream then attaches Symbol.asyncIterator with cleanup (cancel + releaseLock) on done/return/throw.

### NoObjectGeneratedError + error marker pattern
extends AISDKError; marker symbol 'vercel.ai.error.AI_NoObjectGeneratedError'; fields: text?: string, response?: LanguageModelResponseMetadata sans messages, usage?: LanguageModelUsage, finishReason?: FinishReason, plus message/cause; static isInstance via marker. Three documented causes: no response text, unparsable JSON, schema validation failure. Errors carry rich payloads — cannot be plain Zig error codes.

### Usage types + NaN sentinels
LanguageModelUsage: {inputTokens?: number, inputTokenDetails: {noCacheTokens?, cacheReadTokens?, cacheWriteTokens?}, outputTokens?, outputTokenDetails: {textTokens?, reasoningTokens?}, totalTokens?, raw?: JSONObject} (all number|undefined). EmbeddingModelUsage = {tokens: number}. Missing embedding usage defaults to {tokens: NaN}; embedMany sums tokens so NaN propagates. streamObject flush uses NaN placeholders too. A Zig port should replace NaN sentinels with optionals.

### Events/callbacks system
notify({event, callbacks}) awaits all callbacks in parallel via Promise.all, each wrapped in try/catch (silently ignoring failures). Callback<E> = (event:E) => PromiseLike<void>|void. generateObject/streamObject events (all @deprecated but present): GenerateObjectStartEvent (callId, operationId, provider, modelId, prompt/messages, all sampling params, maxRetries, headers, providerOptions, output type, JSON schema, schemaName/Description); StepStart (stepNumber: 0 literal, promptMessages); StepEnd (finishReason, usage, objectText raw string, reasoning, warnings, request, response, providerMetadata, msToFirstChunk); EndEvent<RESULT> (object|undefined, error|undefined — streamObject reports validation errors here instead of throwing). Embed events: EmbedStartEvent/EmbedEndEvent (value: string|string[], embedding: Embedding|Embedding[]) plus per-doEmbed EmbeddingModelCallStart/EndEvent keyed by embedCallId for parallel chunk correlation.

### Model resolution
resolveEmbeddingModel/resolveLanguageModel/resolveRerankingModel: string -> getGlobalProvider().embeddingModel(id) (gateway default); object -> specificationVersion must be in ['v4','v3','v2'] else UnsupportedModelVersionError; older versions adapted via asEmbeddingModelV4/asLanguageModelV4 wrappers. generateObject/streamObject are marked @deprecated in favor of generateText/streamText with an output setting, but remain fully implemented.

### injectJsonInstruction (prompt fallback for non-native-JSON providers)
injectJsonInstruction({prompt?, schema?, schemaPrefix?, schemaSuffix?}): joins [prompt, '', 'JSON schema:', JSON.stringify(schema), 'You MUST answer with a JSON object that matches the JSON schema above.'] with newlines; without schema uses 'You MUST answer with JSON.'. Lives in both ai/src/generate-object and provider-utils (exported for providers to build JSON-mode prompts).

## Port implications
Schema design: the core contract is tiny — (1) a JSON Schema draft-07 document (currently PromiseLike/lazy; in Zig make it a comptime-generated []const u8 or std.json.Value, cached), and (2) an optional validate(allocator, std.json.Value) -> union { ok: T, err: TypeValidationError-like payload }. A comptime schema generator walking @typeInfo (structs -> object with required+additionalProperties:false, optionals -> not-required, enums -> string enum, slices -> array) replaces zod; also offer a runtime jsonSchema() constructor taking a raw JSON-schema string with optional validate fn for the C FFI (Python/Rust wrappers pass pre-serialized JSON Schema + optional callback validator). FlexibleSchema collapses to a tagged union: ComptimeType, RawJsonSchema, NoSchema. DeepPartial is type-level in TS; for Zig either emit a comptime mirror struct with every field optional (feasible via @Type) or expose partials as std.json.Value — the latter matches the SDK (object-mode partials are NOT validated anyway) and is FFI-friendly. ValidationResult/ParseResult must be tagged unions, not error codes, because errors carry payloads (text, response, usage, finishReason in NoObjectGeneratedError); adopt a diagnostics-out-param or result-union pattern throughout. fixJson ports 1:1: linear scan, ArrayList(State) stack, output = input[0..lastValid+1] ++ closers — allocate once. secureJsonParse's proto-pollution defense is JS-specific; std.json needs no equivalent (note and drop). isDeepEqualData runs on every stream delta over the whole accumulated object — port over std.json.Value but consider memoization; NaN token sentinels should become optionals. Concurrency mapping: DelayedPromise -> oneshot future (std.Io ResetEvent + stored value; lazy-construction concern disappears); createStitchableStream -> mutex+condvar FIFO queue of sub-streams; the two TransformStream stages collapse into a single pull-based state machine that reads provider stream parts and yields ObjectStreamPart — natural as a Zig iterator with next(io) for both native use and C FFI (blocking next() or callback registration). streamObject's fire-and-forget IIFE becomes an io.async task owned by the result object; deinit must join/cancel it. embedMany's wave-based Promise.all maps to spawning up to maxParallelCalls (default: unbounded -> use ?usize) concurrent doEmbed tasks per wave with a join barrier; results indexed by chunk to preserve order. Retry is a plain loop with abort-aware sleep honoring retry-after-ms/retry-after headers. Enum output mode maps beautifully to Zig enums (comptime field names -> JSON schema enum, prefix matching for partials). The four output strategies fit a comptime-parameterized generic or a vtable; array-mode textDelta synthesis and last-element-skip logic must be ported exactly to keep textStream emitting valid JSON.
