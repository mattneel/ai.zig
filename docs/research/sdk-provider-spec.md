# sdk-provider-spec

## Summary
The provider spec package (@ai-sdk/provider 4.0.3) currently ships specification version V4 (not V3 — v2/v3/v4 coexist and are all re-exported from src/index.ts; V4 is the newest and what ProviderV4 returns). LanguageModelV4 is a plain object type with specificationVersion 'v4', provider, modelId, supportedUrls (Record<mediaTypePattern, RegExp[]>, sync or PromiseLike), doGenerate(options) -> GenerateResult and doStream(options) -> { stream: ReadableStream<LanguageModelV4StreamPart>, request?, response? }. The stream-part union has 21 tag shapes (text/reasoning/tool-input start-delta-end triples, tool-approval-request, tool-call, tool-result, custom, file, reasoning-file, source with sourceType sub-discriminator url|document, stream-start, response-metadata, finish, raw, error). doGenerate content is a 9-variant union. Usage is a structured object (inputTokens{total,noCache,cacheRead,cacheWrite} / outputTokens{total,text,reasoning}, all number|undefined, plus raw JSONObject); finish reason is {unified: 6-value enum, raw: string|undefined} (v2 used a bare string enum with extra 'unknown'). File payloads use the SharedV4FileData tagged union (data|url|reference|text). Sibling specs: EmbeddingModelV4 (doEmbed), ImageModelV4 (doGenerate), SpeechModelV4 (doGenerate), TranscriptionModelV4 (doGenerate + experimental doStream with its own 8-variant stream-part union), RerankingModelV4 (doRerank), VideoModelV4 (doGenerate; NOT reachable via ProviderV4), FilesV4/SkillsV4 upload interfaces, and RealtimeModelV4 (WebSocket event codec: 8 client-event / 23 server-event unions). ProviderV4 exposes languageModel/embeddingModel/imageModel (required) and transcriptionModel/speechModel/rerankingModel/files/skills (optional). 15 error classes extend AISDKError using a Symbol.for('vercel.ai.error...') marker pattern. JSONValue/JSONObject/JSONArray plus JSONSchema7 (re-exported from the json-schema package) underpin providerOptions/providerMetadata (Record<string, JSONObject>) and tool schemas. Middleware types exist for language/embedding/image models (transformParams + wrap* hooks).

## Key findings
### Current spec version is V4, not V3
package @ai-sdk/provider is 4.0.3; every model interface has readonly specificationVersion: 'v4'. v2, v3, v4 type trees coexist under src/{language-model,embedding-model,...}/{v2,v3,v4}/ and all are exported from src/index.ts (v4 first). The Zig port should target V4 and can ignore v2/v3 except for migration notes.

### LanguageModelV4 interface
Object type: { specificationVersion: 'v4'; provider: string; modelId: string; supportedUrls: Record<string, RegExp[]> | PromiseLike<...> (media-type pattern -> URL-path regexes, matched lowercase; matched URLs are passed natively, not downloaded); doGenerate(LanguageModelV4CallOptions) -> PromiseLike<LanguageModelV4GenerateResult>; doStream(LanguageModelV4CallOptions) -> PromiseLike<LanguageModelV4StreamResult> }. StreamResult = { stream: ReadableStream<LanguageModelV4StreamPart>; request?: { body?: unknown }; response?: { headers?: Record<string,string> } }.

### LanguageModelV4CallOptions (full field list)
prompt: LanguageModelV4Prompt (required); maxOutputTokens?: number; temperature?: number; stopSequences?: string[]; topP?: number; topK?: number; presencePenalty?: number; frequencyPenalty?: number; responseFormat?: {type:'text'} | {type:'json', schema?: JSONSchema7, name?: string, description?: string}; seed?: number; tools?: Array<FunctionTool|ProviderTool>; toolChoice?: {type:'auto'}|{type:'none'}|{type:'required'}|{type:'tool', toolName:string}; includeRawChunks?: boolean (streaming only); abortSignal?: AbortSignal; headers?: Record<string, string|undefined>; reasoning?: 'provider-default'|'none'|'minimal'|'low'|'medium'|'high'|'xhigh' (new in v4); providerOptions?: Record<string, JSONObject>.

### Stream part union — every variant (21 tags)
1) text-start{id, providerMetadata?}; 2) text-delta{id, delta:string, pm?}; 3) text-end{id, pm?}; 4) reasoning-start{id, pm?}; 5) reasoning-delta{id, delta, pm?}; 6) reasoning-end{id, pm?}; 7) tool-input-start{id, toolName, pm?, providerExecuted?:bool, dynamic?:bool, title?:string}; 8) tool-input-delta{id, delta:string, pm?}; 9) tool-input-end{id, pm?}; 10) tool-approval-request{approvalId, toolCallId, pm?}; 11) tool-call{toolCallId, toolName, input: string (stringified JSON), providerExecuted?, dynamic?, pm?}; 12) tool-result{toolCallId, toolName, result: NonNullable<JSONValue>, isError?, preliminary?, dynamic?, pm?}; 13) custom{kind: '{provider}.{type}' string, pm?} (new in v4); 14) file{mediaType, data: FileDataData|FileDataUrl, pm?}; 15) reasoning-file{mediaType, data: FileDataData|FileDataUrl, pm?} (new in v4); 16) source with sourceType:'url'{id, url:string, title?, pm?}; 17) source with sourceType:'document'{id, mediaType, title (required), filename?, pm?}; 18) stream-start{warnings: SharedV4Warning[]} (first part); 19) response-metadata{id?, timestamp?: Date, modelId?}; 20) finish{usage: Usage, finishReason: FinishReason, pm?}; 21) raw{rawValue: unknown} (only when includeRawChunks); 22) error{error: unknown} (multiple allowed mid-stream). pm = providerMetadata?: Record<string, JSONObject>.

### LanguageModelV4Content union (doGenerate output, 9 variants)
text{type:'text', text, pm?}; reasoning{type:'reasoning', text, pm?}; custom{type:'custom', kind:'{p}.{t}', pm?}; reasoning-file{mediaType, data: data|url variants, pm?}; file{mediaType, data: data|url variants, pm?}; tool-approval-request{approvalId, toolCallId, pm?}; source (url|document, same as stream); tool-call (input is stringified-JSON string); tool-result (result: NonNullable<JSONValue>, isError?, preliminary?, dynamic?). GenerateResult = { content: Content[]; finishReason; usage; providerMetadata?; request?: {body?: unknown}; response?: ResponseMetadata & {headers?: Record<string,string>, body?: unknown}; warnings: SharedV4Warning[] (required array) }.

### Usage and FinishReason types
LanguageModelV4Usage = { inputTokens: {total, noCache, cacheRead, cacheWrite: all number|undefined}; outputTokens: {total, text, reasoning: number|undefined}; raw?: JSONObject }. LanguageModelV4FinishReason = { unified: 'stop'|'length'|'content-filter'|'tool-calls'|'error'|'other'; raw: string|undefined }. Contrast: v2 finish reason was a plain string union including 'unknown'; v3/v4 use the structured {unified, raw} object.

### Prompt shape (LanguageModelV4Prompt)
Array<Message>; Message = ({role:'system', content:string} | {role:'user', content:(TextPart|FilePart)[]} | {role:'assistant', content:(TextPart|FilePart|CustomPart|ReasoningPart|ReasoningFilePart|ToolCallPart|ToolResultPart)[]} | {role:'tool', content:(ToolResultPart|ToolApprovalResponsePart)[]}) & {providerOptions?}. TextPart{text}; ReasoningPart{text}; ReasoningFilePart{data: data|url, mediaType}; CustomPart{kind:'{p}.{t}'}; FilePart{filename?, data: SharedV4FileData, mediaType (full type or top-level segment; 'image/*' normalizes to 'image')}; ToolCallPart{toolCallId, toolName, input: unknown (parsed JSON — NOTE asymmetry: generated ToolCall content uses input: string), providerExecuted?}; ToolResultPart{toolCallId, toolName, output: ToolResultOutput}; ToolApprovalResponsePart{approvalId, approved: boolean, reason?}. All parts carry providerOptions?.

### ToolResultOutput union (6 variants)
{type:'text', value:string} | {type:'json', value:JSONValue} | {type:'execution-denied', reason?:string} | {type:'error-text', value:string} | {type:'error-json', value:JSONValue} | {type:'content', value: Array<{type:'text', text} | {type:'file', data:SharedV4FileData, mediaType, filename?} | {type:'custom'}>}. Every variant and nested part carries providerOptions?.

### SharedV4FileData tagged union (replaces v3 untagged DataContent)
{type:'data', data: Uint8Array | string(base64)} | {type:'url', url: URL} | {type:'reference', reference: Record<string,string> & {type?: never} (provider name -> file id)} | {type:'text', text: string}. Model-generated files (LanguageModelV4File/ReasoningFile) restrict to data|url only. v3 used LanguageModelV3DataContent = Uint8Array|string|URL (untagged) — v4's tagged form maps directly to a Zig tagged union. SharedV4ProviderReference deliberately excludes a 'type' key so it can't collide with the tagged shapes.

### Tool definitions and toolChoice
LanguageModelV4FunctionTool = {type:'function', name, description?, inputSchema: JSONSchema7, inputExamples?: Array<{input: JSONObject}>, strict?: boolean, providerOptions?}. LanguageModelV4ProviderTool = {type:'provider', id: '{providerId}.{toolName}', name, args: Record<string, unknown>}. ToolChoice is a 4-variant tagged union (auto/none/required/tool{toolName}).

### SharedV4Warning (4 variants) and provider metadata/options
Warning = {type:'unsupported', feature, details?} | {type:'compatibility', feature, details?} | {type:'deprecated', setting, message} | {type:'other', message}. SharedV4ProviderMetadata and SharedV4ProviderOptions are both Record<string, JSONObject> (outer key = provider name). SharedV4Headers = Record<string,string>. JSONValue = null|string|number|boolean|JSONObject|JSONArray; JSONObject = {[key: string]: JSONValue | undefined}; runtime guards isJSONValue/isJSONArray/isJSONObject in json-value/is-json.ts. JSONSchema7/JSONSchema7Definition re-exported from the 'json-schema' npm package.

### ProviderV4 registry interface
interface ProviderV4 { specificationVersion:'v4'; languageModel(modelId): LanguageModelV4; embeddingModel(modelId): EmbeddingModelV4; imageModel(modelId): ImageModelV4; transcriptionModel?(modelId): TranscriptionModelV4; speechModel?(modelId): SpeechModelV4; rerankingModel?(modelId): RerankingModelV4; files?(): FilesV4; skills?(): SkillsV4 }. Required methods throw NoSuchModelError. NOTE: no videoModel() method exists on any ProviderV* even though VideoModelV4 exists and NoSuchModelError.modelType includes 'videoModel'.

### EmbeddingModelV4
{ specificationVersion:'v4', provider, modelId, maxEmbeddingsPerCall: number|undefined|PromiseLike (Infinity = no limit), supportsParallelCalls: boolean|PromiseLike, doEmbed(options) -> PromiseLike<Result> }. CallOptions = {values: string[], abortSignal?, providerOptions?, headers?: Record<string,string>}. Result = {embeddings: Array<Array<number>> (same order as inputs), usage?: {tokens: number}, providerMetadata?, response?: {headers?, body?}, warnings: Warning[] (required)}.

### ImageModelV4
{ specificationVersion:'v4', provider, modelId, maxImagesPerCall: number|undefined|((opts:{modelId}) => PromiseLike<number|undefined>|number|undefined), doGenerate }. CallOptions (mostly required-but-undefined-able fields, not optional): {prompt: string|undefined, n: number, size: '${w}x${h}'|undefined, aspectRatio: '${w}:${h}'|undefined, seed: number|undefined, files: ImageModelV4File[]|undefined, mask: ImageModelV4File|undefined, providerOptions (required), abortSignal?, headers?}. ImageModelV4File = {type:'file', mediaType, data: string|Uint8Array, providerOptions?} | {type:'url', url: string, providerOptions?}. Result = {images: string[]|Uint8Array[], warnings (required), providerMetadata?: Record<provider, {images: JSONArray} & JSONValue>, response: {timestamp: Date, modelId: string, headers: Record|undefined} (required), usage?: {inputTokens?, outputTokens?, totalTokens?: number|undefined}}.

### SpeechModelV4 and TranscriptionModelV4 (incl. experimental transcription streaming)
SpeechModelV4.doGenerate({text, voice?, outputFormat?, instructions?, speed?: number, language?, providerOptions?, abortSignal?, headers?}) -> {audio: string|Uint8Array, warnings (required), request?{body?}, response: {timestamp: Date, modelId, headers?, body?}, providerMetadata?}. TranscriptionModelV4.doGenerate({audio: Uint8Array|string(base64), mediaType, providerOptions?, abortSignal?, headers?}) -> {text, segments: [{text, startSecond, endSecond}], language: string|undefined, durationInSeconds: number|undefined, warnings, request?{body?: string}, response{timestamp, modelId, headers?, body?}, providerMetadata?}. Optional doStream?({audio: ReadableStream<Uint8Array|string>, inputAudioFormat: {type e.g. 'audio/pcm', rate?}, providerOptions?, abortSignal?, headers?, includeRawChunks?}) -> {stream: ReadableStream<TranscriptionModelV4StreamPart>, request?, response?{timestamp?, modelId?, headers?, body?}}. Transcription stream parts (8): stream-start{warnings}, transcript-delta{id?, delta, pm?} (append-only), transcript-partial{id?, text, startSecond?, durationInSeconds?, channelIndex?, pm?} (revisable), transcript-final{id?, text, startSecond?, endSecond?, channelIndex?, pm?}, response-metadata{timestamp?, modelId?, headers?, body?}, finish{text, segments, language?, durationInSeconds?, pm?}, raw{rawValue}, error{error}.

### RerankingModelV4, FilesV4, SkillsV4, VideoModelV4
RerankingModelV4.doRerank({documents: {type:'text', values: string[]}|{type:'object', values: JSONObject[]}, query: string, topN?, abortSignal?, providerOptions?, headers?}) -> {ranking: [{index (into original list), relevanceScore}] sorted desc, providerMetadata?, warnings? (optional here, unlike others), response?{id?, timestamp?, modelId?, headers?, body?}}. FilesV4 = {specificationVersion:'v4', provider, uploadFile({data: FileDataData|FileDataText, mediaType, filename?, providerOptions?}) -> {providerReference: Record<provider,id>, mediaType?, filename?, providerMetadata?, warnings}}. SkillsV4 = {specificationVersion, provider, uploadSkill({files: [{path, data: FileDataData|FileDataText}], displayTitle?, providerOptions?}) -> {providerReference, displayTitle?, name?, description?, latestVersion?, providerMetadata?, warnings}}. VideoModelV4 = {maxVideosPerCall (like image), doGenerate({prompt, n, aspectRatio, resolution: '${w}x${h}', duration (seconds), fps, seed, image: VideoModelV4File|undefined, frameImages: [{image, frameType: 'first_frame'|'last_frame'}]|undefined, inputReferences, generateAudio: boolean|undefined, providerOptions (required), abortSignal?, headers?}) -> {videos: Array<{type:'url', url, mediaType}|{type:'base64', data, mediaType}|{type:'binary', data: Uint8Array, mediaType}>, warnings, providerMetadata?, response{timestamp, modelId, headers}}}.

### Error taxonomy (15 exported symbols)
Base AISDKError extends Error with {name, message, cause?}; identity via Symbol.for('vercel.ai.error') marker + static isInstance/hasMarker (cross-package-version instanceof). Subclasses each add Symbol.for('vercel.ai.error.AI_<Name>'): APICallError{url, requestBodyValues, statusCode?, responseHeaders?, responseBody?, isRetryable (defaults true for 408/409/429/>=500), data?}; EmptyResponseBodyError; InvalidArgumentError{argument}; InvalidPromptError{prompt} (message prefixed 'Invalid prompt: '); InvalidResponseDataError{data}; JSONParseError{text}; LoadAPIKeyError; LoadSettingError; NoContentGeneratedError; NoSuchModelError{modelId, modelType: 'languageModel'|'embeddingModel'|'imageModel'|'transcriptionModel'|'speechModel'|'rerankingModel'|'videoModel', errorName override}; NoSuchProviderReferenceError{provider, reference}; TooManyEmbeddingValuesForCallError{provider, modelId, maxEmbeddingsPerCall, values}; TypeValidationError{value, context?: {field?, entityName?, entityId?}, static wrap() dedupes identical wraps}; UnsupportedFunctionalityError{functionality}; plus getErrorMessage(unknown) -> string helper.

### Middleware types (language/embedding/image)
LanguageModelV4Middleware = {specificationVersion:'v4', overrideProvider?/overrideModelId?({model}) -> string, overrideSupportedUrls?, transformParams?({type:'generate'|'stream', params, model}) -> PromiseLike<CallOptions>, wrapGenerate?({doGenerate, doStream, params, model}) -> PromiseLike<GenerateResult>, wrapStream?(same args) -> PromiseLike<StreamResult>}. EmbeddingModelV4Middleware adds overrideMaxEmbeddingsPerCall?/overrideSupportsParallelCalls? and wrapEmbed?; ImageModelV4Middleware has overrideMaxImagesPerCall? and wrapGenerate?. Both parametrize on the respective call-options/result types.

### v3 -> v4 delta (for spec-versioning decisions)
v4 additions over v3: (1) reasoning effort enum on CallOptions; (2) custom content part/stream part and CustomPart in prompts (kind: '{provider}.{type}'); (3) reasoning-file content/stream part and ReasoningFilePart in prompts; (4) SharedV4FileData tagged union replacing v3's untagged LanguageModelV3DataContent (Uint8Array|string|URL) and adding reference/text variants; (5) FilesV4/SkillsV4 provider surfaces + SharedV4ProviderReference; (6) mediaType may now be a top-level segment ('image') not just full type. v3 already had: structured usage/finishReason, generate-result/stream-result files, tool approval flow, source document variant. Naming quirks: several v4 files retain v3-era doc comments ('interface version 3'); SpeechModelV4Result references SharedV2Headers (identical shape Record<string,string>).

## Port implications
The whole spec is data + 2-6 method vtables per model type — it maps cleanly to Zig. Model interfaces become vtable structs (ptr + fn pointers taking std.Io) or comptime duck-typed generics; the C ABI layer will want the vtable form. All discriminated unions (stream part, content, prompt parts, file data, warnings, tool choice, tool-result output, realtime events) become tagged unions; the discriminators are string literals in JSON, so each union needs a stable tag-name <-> enum mapping for (de)serialization, and the C ABI needs a parallel enum + per-variant struct layout. Hard/design-sensitive points: (1) supportedUrls uses RegExp[] — Zig std has no regex; either embed a small regex engine, restrict to literal-prefix/glob matching, or expose a match-callback in the vtable. (2) ReadableStream<StreamPart> maps to a pull-based iterator (next() !?StreamPart) built on Zig 0.16 std.Io streaming; doStream returns {stream, request, response} so the response headers must be available before the first part. (3) PromiseLike-typed properties (supportedUrls, maxEmbeddingsPerCall, supportsParallelCalls) should become plain values or resolver fns — Zig's std.Io makes them synchronous calls that may block. (4) AbortSignal maps to std.Io cancellation. (5) JSONValue/JSONObject need an arena-allocated JSON value type (std.json.Value works); JSONObject permits undefined values — collapse to absent keys. (6) number|undefined everywhere -> optionals (?u64 for tokens, ?f64 for sampling params); Date -> i64 epoch millis. (7) Error marker pattern -> Zig error set + side-channel detail struct (error payloads like APICallError's statusCode/responseBody can't ride Zig errors; use a diagnostics out-param, and expose codes + getters over the C ABI). (8) Template-literal types ('{provider}.{type}', '{w}x{h}') are just strings with validation helpers. (9) Watch the tool-call input asymmetry (string in generated content, parsed unknown in prompt parts). (10) RealtimeModelV4 is a pure event codec over WebSocket — port it as parse/serialize functions over the tagged unions; transport lives elsewhere.
